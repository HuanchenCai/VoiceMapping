# VoiceMap UI 设计方案选项

> 4 种风格的备选方案。每个都是完整的设计方向，不是"哪个组件改改"。
> 选定一个之后，把 `UI_DESIGN.md` 替换成那个方向的细则。

---

## 共通约束（不论选哪个方案都要满足）

- tkinter + matplotlib 实现可行
- 中英文双语切换（A0-4）
- 主窗口最小 1200×720
- VRP 热图始终是中心元素，不能被装饰品挤到边角
- 软著截图能体现"专业感"和"独立软件"特征
- 主字体 Microsoft YaHei UI（汉字/西文都有专门字形）

---

# 方案 A — Modern Dashboard（现代 SaaS 风）

## 定位

一句话：**"研究者的 Linear / Datadog"**。年轻、现代、信息密度适中、视觉有呼吸感。

参照：[Linear](https://linear.app) · [Datadog](https://datadoghq.com) · [Notion](https://notion.so) · GitHub Dark · Vercel

## 调性

- 深色为主（#0a0e13 → #131922 → #1a212d 三层）
- 单一强调色 cyan #00d9ff（信号感强，跟 VRP 热图配色不冲突）
- 圆角 8-12px，subtle border 1px
- 中等留白（8/16/24/32 间距栅格）
- 字体：Microsoft YaHei UI 13-15px，主标题 18px

## 布局

```
├──────┬──────────────────────────────────────────┬──────────┤
│      │ 指标 [声学] [EGG] [唱歌] [聚类] [更多▾] │          │
│ 文件 │ ┌────────────────────────────────────┐  │ 详情     │
│      │ │                                    │  │          │
│ ✓    │ │      VRP Heatmap                   │  │ CPP      │
│ test │ │                                    │  │ 16.79 ✓ │
│      │ │  ◀                            ▶   │  │          │
│ ○    │ │                                    │  │ 临床范围 │
│ rec_2│ │                                    │  │ ≥14 良好 │
│      │ └────────────────────────────────────┘  │ 10-14… │
│ ⊕    │ [拟合▾] [标注] [复位] [复制] [保存▾]   │          │
│      │                                          │ [导出]   │
├──────┴──────────────────────────────────────────┴──────────┤

└────────────────────────────────────────────────────────────┘
```

## 优 / 劣

✅ 截图传播好看，社交媒体友好  
✅ 视觉跟 GitHub / VS Code 同代沟通成本低  
✅ Dark mode 配 VRP 热图天然搭配  
✅ Inspector 右栏放临床范围信息很自然  
⚠️ 信息密度比纯研究工具略低（适合"看一个 metric"的工作流，不适合"同时看 20 个"）  
⚠️ 软著审查员可能觉得"看起来像玩具/网页"（深色 + cyan 不像传统科研软件）  
❌ 论文里截图风格突兀（学术期刊偏好 light theme）

## 适合谁

- 个人研究者 / 工程师
- 软著申请人是科技公司风格
- 给 VC / demo 用
- 不适合：医院临床、纸面论文 figure

# 方案 B — Clinical Workstation（专业医疗软件风）

## 定位

一句话：**"嗓音诊室的 Tableau / Epic"**。冷峻、密集、可信、像三甲医院在用的专业系统。

参照：Praat 表格视图 · MATLAB · Tableau Health · Epic / Cerner · MicroLab 6500 · KayPENTAX CSL

## 调性

- 浅色为主（#f8fafc → #ffffff → #f1f5f9）
- 强调色 navy #1e40af 或 teal #0891b2（医疗常用色）
- 几乎无圆角（0-2px），1px hairline border
- 信息密度高，间距小（4/8/12/16）
- 字体：Microsoft YaHei UI 12-13px，标题 14px Bold
- 表格 / 数字 / 标签为主，按钮少

## 优 / 劣

✅ 软著审查员认得这种 layout（"严肃专业软件"）  
✅ 一屏看 N 个 metric 数值表（临床想看的"全 panel 数据"一目了然）  
✅ 浅色截图直接进论文不用调  
✅ 临床用户上手快（习惯 Epic / 国产 HIS 系统的人）  
⚠️ 视觉不"现代"，年轻用户可能觉得过时  
⚠️ 浅色 + 高密度容易视觉疲劳  
❌ Dark mode 必须做反色，工作量翻倍

## 适合谁

- 临床嗓音治疗师 / 喉科医生
- 教学医院、专科诊所
- 软著审查员（这是**最像传统医疗软件**的方案）
- 不适合：声乐爱好者 / 自媒体演示

# 方案 C — Studio（录音棚 / Audition 风）

## 定位

一句话：**"嗓音的 Adobe Audition / Reaper"**。深沉、专业音乐工具感、多文件横向并列、强调"工作流"而非"诊断"。

参照：Adobe Audition · Reaper · iZotope RX · Ableton Live · DaVinci Resolve

## 调性

- 极深色（#0a0a0a → #1a1a1a → #2a2a2a 灰阶）
- 强调色 amber #f59e0b 或 orange #ea580c（"录音灯"色）
- 直角或 2-4px 圆角
- 信息密度高（多 panel 并列）
- 字体：Microsoft YaHei UI 12-13px
- "tracks" / "transport" / "mixer" 隐喻

## 优 / 劣

✅ 多文件管理是一等公民（M5.5 文件管理面板天然契合）  
✅ 歌手 / 录音师用户秒上手（视觉熟悉）  
✅ "tracks" 概念支持后续多文件叠加 metric 的扩展  
✅ amber 配色辨识度高，跟其他方案差异化大  
⚠️ "诊断 / 临床" 调性弱（amber 暖色不像医疗软件）  
⚠️ 软著审查员可能觉得"这是音乐软件不是嗓音软件"  
❌ 跟 VRP 数据本质（grid，不是 timeline）有错位（隐喻可能误导）

## 适合谁

- 声乐教师、歌手训练
- 配音 / 播音从业者
- 录音工程师做嗓音质控

# 方案 D — Academic / Paper-First（学术发表风）

## 定位

一句话：**"截图直接进期刊"**。极简、印刷友好、发表第一、屏幕第二。

参照：Mathematica 文档 · LaTeX Beamer · Nature/Science figure · TidyverseR · Jupyter notebook 输出

## 调性

- 纯白底 #ffffff，黑字 #1a1a1a
- 强调色深色（藏青 #1e3a5f 或 勃艮第 #7c2d3e），不用亮色
- 几乎无装饰，直角，hairline 0.5-1px border
- 大量留白
- 主字体可选 serif（Source Han Serif / 宋体）做标题，正文 sans
- 数字用等宽（Cascadia Code / Consolas）
- 不带 dark mode（学术不需要）

# 我的推荐（仅供参考）

按你的实际场景排：

### 软著申请人 + 个人研究者 + 写论文的预期 → **方案 B**

理由：
- 软著审查员看着像专业软件（10 分）
- 浅色截图直接进论文（9 分）
- 临床能用、学术能用，覆盖最广
- 实现难度只比"最简单的 D"高一档
- 中英切换在浅色高密度 layout 上视觉不会乱

### 如果你优先 demo / 社交传播 → **方案 A**

理由：
- 现代感最强，对外宣传/社交媒体晒图好看
- 视觉跟 GitHub / VS Code 同代，年轻用户立刻 get
- 但软著审查员可能觉得"花哨"

### 如果你想押宝歌手市场 → **方案 C**

理由：
- 录音棚视觉是这个用户群的母语
- M5.5 文件管理天然契合 tracks 隐喻
- 但学术 / 临床场景不友好

### 如果你纯学术 / 不打算上市 → **方案 D**

理由：
- 截图就是论文 figure
- 实现成本最低
- 但作为软件软著申请，密度可能太低

# 折中：**方案 B + A 的混血**

如果你不想 100% 押任何一个，最务实的是：

> **基础架构按 B 走（浅色 + navy + 高密度 + 临床气场），但借鉴 A 的现代细节：圆角 8px、subtle hover、accent glow、status pills**

这样：
- 浅色背景 → 软著截图、论文、印刷都友好
- 高密度 layout → 临床认可
- 圆角 + hover → 不显得过时（不像 1998 年的 Win32）
- Status pill / accent glow → 现代感不拉胯

副作用：要做 light theme 为主，dark theme 为辅（M4 阶段）。
