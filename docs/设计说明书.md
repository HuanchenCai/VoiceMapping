## 4. 数据流

### 4.1 单文件分析（GUI）

```
用户拖入 wav
   ↓
gui.app._on_drop → _on_audio_dropped(path)
   ↓
_tracks_add(path) → 创建 TrackEntry，读 wav 头（sr/duration/channels）
   ↓
_tracks_set_active(idx)  ← 第一个文件自动 active
   ↓
_start_analysis(path)  ← active 触发分析
   ↓
worker thread:
  analyzer.analyze_and_output_vrp(audio, ...)
    1. load_audio              soundfile 读 wav，分声/EGG 通道
    2. preprocess              EGG 带通 (FonaDyn type=3) + 高通 + 压缩扩展
    3. detect_cycles           Phase Portrait + numba JIT，输出 cycle 起止
    4. filter_cycles           clarity 阈值过滤
    5. calculate_all_metrics   每个 calculator 跑过 cycle_triggers
        ├── SPL / Clarity / CPP / SpecBal / Crest / Entropy
        ├── Qcontact / Icontact / dEGGmax / HRFegg / OQ / SPQ / CIQ
        ├── Jitter (3 种) / Shimmer (5 种) / HNR / NHR
        ├── Vibrato rate/extent/jitter
        ├── Formant F1/F2/F3 + bandwidth + dispersion
        ├── EGG K-means cluster (5 簇) + cPhon K-means (5 簇)
        └── M1: RMS / F0_Hz / Spectral moments / MFCC×13 / 等
    6. partial_cb              第一组指标完成 → 主线程渲染早期热图
    7. csv_writer.write_vrp    构 DataFrame → range filter → groupby agg
                               → 空簇救援 → cluster 列聚合 → CSV 写盘
    8. plotter.plot_vrp_*      可选 PNG 导出
   ↓
回主线程（msg_q → _drain_queue 'done'）
   ↓
更新 active TrackEntry.df / state="analyzed"
   ↓
_render_metric / _update_inspector / _update_statusbar
```

### 4.2 鼠标 hover 探测

```
matplotlib motion_notify_event
   ↓
_on_canvas_motion(event)
   ↓
event.xdata=MIDI, event.ydata=SPL → round to int
   ↓
_last_df 查 cell  → metric value
_THRESHOLDS 查严重度档位
   ↓
_update_inspector_value() → .configure() 4 个 Label（不重建 widget）
```

### 4.3 多文件切换

```
用户点 Tracks Panel 第 N 行
   ↓
_tracks_set_active(N)
   ↓
TrackEntry[N].state == "analyzed"?
  yes → 把 TrackEntry.df 同步到 self._last_df / audio_path / 等
        → _render_metric / _update_inspector / _update_statusbar
  no  → _start_analysis(TrackEntry.path)
```

## 5. 关键模块

### 5.1 voicemap.config

`@dataclass class VoiceMapConfig`：所有可调参数集中。`DEFAULT_CONFIG`
作为单例。GUI 设置面板修改的是这份；analyzer 启动时传入。

主要字段：sample_rate, clarity_threshold, n_min/max_midi, n_min/max_spl,
spl_correction_db, output_dir, audio_file。

### 5.2 voicemap.analyzer

`class VoiceMapAnalyzer`：分析编排。约 19 个方法、660 行；
CSV 输出逻辑独立在 `csv_writer.py`，渲染独立在 `plotter.py`。

主要方法：
- `analyze_and_output_vrp(audio_path, return_df, plot_mode, progress_cb,
   partial_cb)` — 顶层入口
- `load_audio` / `preprocess_egg` / `phase_portrait_cycle_detection` /
  `filter_cycles` / `calculate_all_metrics`
- `train_cluster_centroids(wavs)` — 多 wav 联合 K-means
- `load_centroids(path)` / `save_centroids(path)` — cEGG.csv I/O
- `_aggregate_cluster_labels` — maxCluster + share% 列

### 5.3 voicemap.metrics

20+ 个 calculator 类，每个都遵循 `__init__(config) + calculate(signal, ...)`
约定。组织（每个一段）：

| 组 | 类 | 输出列 |
|----|-----|--------|
| 基础 | SPLCalculator / ClarityCalculator | spl / clarity |
| 频谱 | CPPCalculator / SpecBalCalculator / CrestCalculator | cpp / specbal / crest |
| EGG | QcontactCalculator / EntropyCalculator / HRFCalculator | qcontact / entropy / hrf 等 |
| 周期 | OpenQuotientCalculator | oq / spq / ciq |
| 扰动 | PerturbationCalculator | jitter, shimmer 各种 |
| 噪声 | HNRCalculator / NHRCalculator | hnr / nhr |
| 唱歌 | VibratoCalculator / FormantCalculator / FormantExtrasCalculator / HarmonicDiffCalculator | vibrato_*, f1/f2/f3, b1/b2/b3, h1h2 等 |
| 聚类 | ClusterCalculator / PhonClusterCalculator | cluster / phon (1-5) |
| M1 | SpectralMomentsCalculator / F0HzCalculator / MFCCCalculator / IntegrativeMetricsCalculator / VibratoJitterCalculator / NHRCalculator / CPPSCalculator / PPECalculator / ZCRCalculator | rms, spec_*, mfcc1-13, alpha_ratio, hammarberg, gne, mpt, voicing_ratio, duv, vib_jitter |

#### 关键算法决策

- **EGG 周期检测**：相位肖像（Dolansky 算法）+ numba JIT，~2.4× 速度
- **Formant 带宽**：LPC 频谱 -3dB FWHM（取代 np.roots()，27s → 2.8s），
  邻峰约束防止跨过去；800 Hz 硬上限
- **K-means 空簇救援**：3 层（UI / Calculator 内 / Aggregator 后）确保
  每个 (1..k) 标签都至少出现一次
- **MFCC**：自实现 Mel 滤波器 + DCT（不依赖 librosa）
- **Vibrato**：F0 时序 → bandpass 3-10 Hz → 过零 + autocorr

完整 calculator 实现见 `voicemap/metrics.py`，每个类有 docstring。

### 5.4 voicemap.metrics_registry

`@dataclass class MetricSpec`：metric 元数据登记。每个有效输出列必须
有 spec：`key, category, label, vmin, vmax, unit, cmap, description`。
`get(key) -> Optional[MetricSpec]`。

GUI Inspector 的"描述 / 单位"读这个；plotter 的 vmin/vmax 也读这个。
新加 metric 改一处即可。

### 5.5 voicemap.csv_writer

`write_vrp(analyzer, metrics, return_df, plot_mode, export_plots, write_disk)`
—— 从 analyzer 拆出来的 252 行写盘逻辑。流程：

1. 对齐所有 calculator 输出到 `base_n`（cycle 数）
2. 构 DataFrame（每行一个 cycle，82 列）
3. range filter（MIDI ∈ [30, 96]，SPL ∈ [40, 120]）
4. groupby (MIDI, dB) agg（Clarity 取 max，其他取 mean，Total 取 sum）
5. 空簇救援
6. maxCluster + Cluster {1..5} 列聚合（同 cPhon）
7. 标准列序排列（83 列）
8. 写 CSV（分号分隔）+ plot 分发

### 5.6 voicemap.plotter

`draw_vrp_on_ax(ax, fig, df, col)`：单个 metric 的 (MIDI × SPL) heatmap
渲染。`plot_vrp_dataframe(df, ts_base, plot_dir)`：批量 PNG 导出。
`plot_vrp_combined(df, ts_base, plot_dir)`：单图合并所有 metric 的总览。

颜色映射 11 种 cmap，沿用 KTH FonaDyn 的 HSV/RGB 公式（Metric*.sc 文件）。

### 5.7 voicemap.report

临床叙述报告。核心是 `_THRESHOLDS: Dict[str, List[Tuple[lo, hi, label, severity]]]`
查询表：每个 metric 一组档位，severity ∈ {good, normal, watch, abnormal}。

`generate_report(grouped_df, out_path, audio_name)` —— 6 大类：总览 /
嗓音质量 / EGG / 唱歌 / 频谱 / 自动观察。带跨指标启发式（如 Jitter 高 +
Shimmer 高 → "嗓音粗糙"）。

GUI Inspector 的临床范围卡也读 `_THRESHOLDS`。

### 5.8 voicemap.gui.app

`class VoiceMapApp(_TkBase)`：主窗口。约 2700 行；
自定义控件、对话框、工具栏拆到 `gui/widgets.py`、`gui/dialogs.py`、
`gui/modern_menu.py`、`gui/theme.py`。

主要职责：
- `_build_*` 系列：菜单栏 / 标题 / Tracks Panel / Metric Bar / Inspector / Status Bar
- 5 个 popup factory：`_popup_file/edit/metric/view/help`（菜单点开时调）
- 多文件状态：`_tracks: list[TrackEntry]` + `_active_track: int`
- 分析线程：`_start_analysis` 走 worker thread，msg queue 送回主线程
- i18n 订阅：`_on_language_changed` 重建菜单 + 更新 persistent widgets

### 5.9 voicemap.gui.modern_menu

自画 menubar + popup（绕开 tk.Menu 在 Win11 上的老 Win32 USER 白边）。
`ModernMenubar` 是 horizontal Frame of label-buttons；`ModernPopup` 是
borderless Toplevel + 自定义 row layout。Win32 `SetWindowRgn` 给 popup
加圆角。

### 5.10 voicemap.gui.theme

颜色 + 字体 + METRIC_SECTIONS 单一来源。option-C 调色板（深灰 + amber）：


## 6. 关键设计决策

### 6.1 为什么不用 PyQt？

CLAUDE.md 里明确：tkinter + sv-ttk + 自画菜单已经够用，切换 GUI
框架不带来研究价值，反而增加打包体积、依赖、平台兼容问题。

### 6.2 为什么自画 menu / popup？

`tk.Menu` 在 Win11 上画 1px 白边（老 Win32 USER 菜单 API 限制，与
DWM Fluent 渲染管线没有 binding）。视觉接近现代应用要求 Toplevel +
overrideredirect 自画 popup，Win32 `SetWindowRgn` 加圆角。

### 6.3 为什么 K-means 空簇要 3 层救援？

sklearn KMeans 偶尔产生空簇。VRP 输出要求每个 (1..k) 标签至少出现
一次（否则 maxCluster 显示有缺口，cPhon 也是）：
1. **UI 层**：metric 下拉始终列 Cluster 1..k，即使空也保留入口
2. **Calculator 层**：空簇就把"最差拟合"的 cycle 重赋
3. **Aggregator 层**：post-filter 后还空，强行抢 1 个点填进去

### 6.4 为什么 EGG 公式不改成 Praat 风格？

Qcontact / Icontact / HRFegg / Entropy 跟 KTH FonaDyn (SC 版) 完全一致
是本软件的身份标识。要 Praat 风格的用户该直接用 Praat。

### 6.5 为什么走 dict-based i18n 而不是 gettext？

软件只有 ~160 个用户字符串，dict 性能足够（O(1) 查找），无 .po/.mo
编译麻烦，全 UTF-8 源文件 grep-friendly。

### 6.6 为什么把 log 移到独立窗口？

option-C 设计 spec 不在 Inspector 放 log；保持 Inspector 专注当前
metric 详情。日志通过 视图 → 日志面板 弹独立 Toplevel 看。`_append_log`
保留兼容旧 call site，但底层 `tk.Text` 不 pack（不可见）。

## 7. 性能

| 操作 | 耗时（60s 立体声 wav，i7-12700） |
|------|----------------------------------|
| 音频加载 + 预处理 | ~0.3 s |
| 周期检测 (numba) | ~0.5 s |
| 全 metric 计算 | ~10 s |
| CSV 写盘 + plot 分发 | ~1.5 s |
| **总耗时** | **~12.5 s**（5.6× 实时） |
| GUI 启动到可交互 | ~1.5 s |
| 第一次 numba 编译 | +1-2 s（仅首次） |

内存峰值 ~800 MB（matplotlib + 所有 cycle features）。

## 8. 测试与验证

`tests/validate_params.py`：4 维度回归（Praat 交叉对照 / 范围检查 /
结构检查 / 内部一致性）。当前基准 **48 PASS / 4 WARN / 0 FAIL**：

- WARN 4 项：Jitter / JitterRAP / JitterPPQ5（EGG-周期切分 vs Praat
  voice-autocorr 切分的方法学差距，已知 ~3-4× 偏高，文献确认）；
  maxCPhon 缺标签 4（K-means 簇 4 偶尔空，三层救援已处理）。

每次结构性变动 / 算法变动 commit 之前必须跑过 validate_params.py。

## 9. 部署 / 打包

PyInstaller 单文件打包：

```bash
pyinstaller --onefile --windowed \
    --name VoiceMap \
    --icon assets/voicemap.ico \
    --hidden-import="numba.core.types" \
    --collect-all sv_ttk \
    --collect-all matplotlib \
    --collect-all soundfile \
    --add-data "voicemap;voicemap" \
    voicemap/cli.py
```

输出 `dist/VoiceMap.exe`，预期 < 200 MB。Inno Setup 包装为 setup.exe。

详见 `pyproject.toml` 的 `[project.scripts] voicemap = "voicemap.cli:main"`
入口配置。
