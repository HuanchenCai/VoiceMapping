# VoiceMap — Methodology & Algorithm Reference

> 嗓音声学品质多维分析图谱 (VoiceMap) — per-metric algorithm, formula,
> literature citation, and validation status.

VoiceMap analyses a voice (and, when available, a synchronous EGG channel)
and produces a **two-dimensional Voice Range Profile (VRP)**: every analysed
glottal cycle is binned into a `(MIDI semitone × dB SPL)` grid cell, and each
cell carries a battery of per-cell acoustic / electroglottographic / singing /
clustering metrics (~64 CSV columns in total). A cell's value is the
aggregate (mean, max, or sum, per the registry) of the per-cycle values that
fall in that bin.

This document is the **algorithm reference**: for each metric it states what
the metric measures, the exact algorithm / formula as implemented, the
literature it derives from, and its validation status. The single source of
truth for the metric set is `voicemap/metrics_registry.py`; every metric below
matches a `MetricSpec` registered there. Where a metric has a full validation
document under `docs/validation/metrics/`, this reference links to it and
summarises only the headline result — the validation doc carries the complete
8-section record (implementation, reference standard, test signals, method,
results, status, limitations, change log).

**Provenance note.** Several spectral-shape, EGG, and clustering metrics were
ported from the project's predecessor, a SuperCollider Voice Range Profile
implementation; throughout this document that origin is referred to as **"the
reference SuperCollider VRP implementation"**. Where a formula is a VoiceMap-
specific convention (rather than a textbook standard), this is stated
explicitly.

## Validation methodology

Every metric is validated by at least one of three evidence types (defined in
`docs/validation/PLAN.md`):

- **(A) Numerical parity** — the output matches an authoritative reference
  tool (Praat, librosa, OpenSMILE, COVAREP/VoiceSauce, MDVP, scikit-learn,
  `nolds`) to a stated tolerance, usually by feeding both implementations the
  *same* intermediate input (e.g. the same cycle marks or the same
  spectrogram) so that only the formula layer is under test.
- **(B) Synthetic ground truth** — a signal with a known answer (imposed
  jitter, a sinusoidally-modulated F0, a two-tone band ratio, a synthetic EGG
  with a known closed quotient) is run through the calculator and the recovered
  value is checked against the closed form.
- **(C) Real-corpus behaviour** — on a public corpus, the output distribution
  agrees with the literature (e.g. a healthy-vs-pathological separation).

The acceptance bar by priority tier: **P0** metrics target A + B + C; **P1**
metrics require at least A or B; **P2 / experimental** metrics require at least
B or C. A metric flagged `待验证` in the registry is computed and written to
CSV but has not yet cleared cross-validation; this is noted per metric below.

The synthetic signals live in `docs/validation/test_signals/` (generated by
`make_signals.py`); the per-metric harness is `scripts/validate_metric.py
<name>`, which writes the auto-generated Results table into each validation
doc.

## Acoustic

Frame- or cycle-level descriptors of the radiated voice signal: periodicity,
noise, spectral shape, and the clinical perturbation family.

### Clarity (`Clarity`, unitless)
NSDF pitch-detection confidence — the strength of periodicity at the detected
pitch, and the VRP's primary quality gate (cells are kept at `Clarity ≥ 0.96`).
Computed by a McLeod–Wyvill Normalised Square Difference Function:
`NSDF(τ) = 2·Σ x[j]·x[j+τ] / Σ (x[j]² + x[j+τ]²) ∈ [−1, 1]`; **Clarity** is the
NSDF value at the selected pitch lag (≈ 1 for a clean periodic voice). Pipeline:
30 Hz HPF → leaky integrator (`a = 0.995`) → 2048-pt windows, hop 1024 → NSDF
→ global-argmax with iterative octave correction → subharmonic repair.
*Reference:* McLeod & Wyvill (2005), *"A Smarter Way to Find Pitch"*, ICMC;
constants match the reference SuperCollider VRP implementation's Tartini UGen
(no Praat equivalent — Clarity is the NSDF peak height, not Praat pitch
strength).
*Validation:* **PASS** — (B) synthetic octave-stress: clean-vowel Clarity 0.999,
breathy < clean. [f0_clarity.md](validation/metrics/f0_clarity.md)

### CPP (`CPP`, dB)
Cepstral Peak Prominence — prominence of the cepstral peak at the fundamental
quefrency above the cepstral baseline; a robust correlate of dysphonia /
breathiness needing no F0 tracking. Per 100 ms Hann window: `rfft` of 2048 →
**natural-log** magnitude → 1024-pt IFFT of the first 1024 log-mag bins →
`|cepstrum|` → dB; least-squares regression line over the search band
(quefrency↔frequency `f = SR/2q`, bins `[25, 367]` ↔ `[882, 60] Hz`); **CPP =
max residual above the regression line**.
*Reference:* Hillenbrand, Cleveland & Erickson (1994) / Hillenbrand & Houde
(1996); Praat `PowerCepstrogram → CPPS` (Maryn et al. 2010). Uses the reference
SuperCollider `Cepstrum` natural-log convention — **not** numerically equal to
Praat CPPS.
*Validation:* **PASS** (ordering / correlation, per CPP's convention-dependence)
— (A) `r = 0.98` vs Praat CPPS over an SNR sweep, (B) `r = 0.99` monotone in
SNR. [cpp.md](validation/metrics/cpp.md)

### CPPS (`CPPS`, dB)
Smoothed CPP (Hillenbrand 1996 / Maryn et al. 2010 smoothed variant) — the
time- and quefrency-smoothed sibling of CPP, sharing the same cepstral pipeline
and convention. Validated together with CPP (same convention-dependent scale,
not Praat-equal in absolute value).
*Reference:* Hillenbrand & Houde (1996); Maryn et al. (2010).
*Validation:* **PASS** (alongside CPP). [cpp.md](validation/metrics/cpp.md)

### SpecBal — Spectrum Balance (`SpecBal`, dB)
High-band-minus-low-band level — a spectral-tilt / brightness measure related
to vocal effort and mode. Low band = 4th-order Butterworth `BLowPass4 @ 1500 Hz`,
high band = `BHiPass4 @ 2000 Hz` (`rq = 1.4`); per band a sliding RMS → dB;
`SpecBal = hi_dB − lo_dB`, clipped to ±50 dB. (The registry description
`10·log10(E_below_1500 / E_above)` is the conceptual band ratio; the
implementation realises it as the Butterworth-band dB difference.)
*Reference:* the reference SuperCollider VRP implementation (`BLowPass4` /
`BHiPass4` + RMS dB); no external Python tool computes this exact band split.
*Validation:* **PASS** — (B) two-tone GT: SpecBal linear in the imposed band
ratio (slope 1.0002, `r = 1.00`), constant +0.41 dB filter offset.
[specbal.md](validation/metrics/specbal.md)

### Crest factor (`Crest`, unitless)
Peak-to-RMS ratio within each glottal cycle — a "peakiness" measure (pressed /
clear phonation is peakier, breathy / noisy flatter). Per cycle `[s, e)`:
`Crest = max(|x|) / RMS(x)`, after a 20 ms voice delay that aligns the acoustic
cycle with its EGG trigger.
*Reference:* standard definition (Baken & Orlikoff 2000); analytic per-shape
constants — sine `√2`, square `1`, sawtooth `√3`.
*Validation:* **PASS** — (B) sine 1.4158, square 1.0000, sawtooth 1.7301, all
< 0.2 % from the analytic constant. [crest.md](validation/metrics/crest.md)

### Sample Entropy / CSE (`Entropy`, unitless)
Sample entropy of the cycle-to-cycle variation of the EGG harmonic structure
(amplitude + phase) — a nonlinear regularity measure (irregular / disordered
phonation → higher entropy). Per EGG cycle, DFT → harmonic amplitudes (in Bel,
`2·log10|·|`) and phases; per harmonic a sliding window → **m = 1 Sample
Entropy** `−ln(A/B)` (`B` = count of length-1 template pairs within tolerance
`r`, `A` = the subset also matching at the next sample); summed across harmonics
into one entropy per cycle.
*Reference:* Richman & Moorman (2000), *Am. J. Physiol.* Reference tool
`nolds.sampen` (`emb_dim=1`).
*Validation:* **PASS** — (A) the m=1 primitive is byte-identical to `nolds`
(max |Δ| 0 over noise / sine / AR(1)); (B) disorder ordering ramp < sine <
noise. [cse.md](validation/metrics/cse.md)

### Jitter — local / RAP / PPQ5 (`Jitter`, `JitterRAP`, `JitterPPQ5`, %)
Cycle-to-cycle variation of the glottal **period** (MDVP / Praat frequency-
perturbation family). For valid periods `T_i` between consecutive cycle marks:

```
Jitter (local) = mean_i |T_i − T_{i−1}|            / mean_i T_i
JitterRAP      = mean_i |T_i − mean(T_{i−1..i+1})| / mean_i T_i
JitterPPQ5     = mean_i |T_i − mean(T_{i−2..i+2})| / mean_i T_i
(JitterDDP     = 3 · RAP, by definition)
```

A period is included only if it passes the Praat `is_period` gate (length in
`[pmin, pmax] = [1e-4, 0.02]`, not a both-sides outlier at
`max_period_factor = 1.3`). Cycle marks come from `praat_pitch.py`
(`PointProcess (cc)`).
*Reference:* Praat `VoiceAnalysis.cpp` (`PointProcess_getJitter_*`); MDVP;
Baken & Orlikoff (2000). Compared against `parselmouth`.
*Validation:* **PASS** — (A) parity vs Praat on real cycle marks `Δ = 0` at
`atol 1e-9`; (B) synthetic GT recovers imposed 0.5 % / 2 % / 0 % exactly.
[jitter.md](validation/metrics/jitter.md)

### Shimmer — local / APQ3 / APQ5 / APQ11 / dB (`Shimmer`, `ShimmerAPQ3`, `ShimmerAPQ5`, `ShimmerAPQ11`, %; `ShimmerDB`, dB)
Cycle-to-cycle variation of the glottal **amplitude** (MDVP / Praat amplitude-
perturbation family). For accepted-pulse amplitudes `a_i` (each from a Hann-
windowed per-period RMS amplitude tier):

```
Shimmer (local) = mean_i |a_i − a_{i−1}|            / mean_i a_i
ShimmerDB       = 20 · mean_i |log10(a_i / a_{i−1})|             [dB]
ShimmerAPQn     = mean_i |a_i − mean(a_{i−h..i+h})| / mean_i a_i   (n = 2h+1)
(ShimmerDDA     = 3 · APQ3, by definition)
```

A pulse contributes only if its periods are in `[pmin, pmax]` and adjacent
amplitude ratios are `≤ max_amplitude_factor = 1.6`.
*Reference:* Praat `AmplitudeTier.cpp` (`AmplitudeTier_getShimmer_*`) and the
per-period Hann-RMS amplitude tier; MDVP; Baken & Orlikoff (2000).
*Validation:* **PASS** — (A) amplitude-tier identity (pulse count + times +
values) then formula parity vs Praat, `atol 1e-6`; (B) synthetic GT recovers
imposed 5 % and its closed-form dB exactly. [shimmer.md](validation/metrics/shimmer.md)

### HNR (`HNR`, dB)
Harmonics-to-Noise Ratio — ratio of periodic to aperiodic energy; the headline
measure of breathiness / hoarseness. Praat-style autocorrelation HNR on Hann-
windowed 40 ms frames, 10 ms hop, pitch search `[60, 400] Hz`: per frame
`p = max_τ r(τ)` (window-autocorrelation-compensated, `r(0)`-normalised),
`HNR = 10·log10(p / (1 − p))`, with the peak clipped to a **40 dB ceiling**.
*Reference:* Praat `To Harmonicity (cc)`; Boersma (1993), IFA Proceedings 17.
Physical anchor: for signal + white noise at SNR `S` dB,
`HNR_dB = 10·log10(p/(1−p)) = S` exactly.
*Validation:* **PASS** — (B) HNR == imposed SNR to < 0.3 dB over 5–25 dB;
(A) Praat parity < 0.3 dB on stationary vowels. (Note: ~6 dB optimistic vs
Praat on non-stationary running voice — see doc §7.)
[hnr.md](validation/metrics/hnr.md)

### NHR (`NHR`, unitless) — `待验证`
Noise-to-Harmonics Ratio — the linear (MDVP-style) inverse of HNR.
`NHR = 1 / H`, where `H = 10^(HNR_dB/10)` is the linear HNR; a thin per-cycle
inverse of HNR. Pathology cut ≈ 0.19 (≈ HNR 7.2 dB).
*Reference:* MDVP Noise-to-Harmonics Ratio; Baken & Orlikoff (2000).
*Validation:* **PASS** (validated within the HNR doc) — (B) `NHR == 1/H_linear`
to `atol 5e-3`. (Marked `待验证` in the registry; the inverse identity is
verified in hnr.md.) [hnr.md](validation/metrics/hnr.md)

### PPE — Pitch Period Entropy (`PPE`, unitless) — `待验证`
Entropy of the cycle-to-cycle pitch-period variation after de-trending — a
nonlinear dysphonia marker (disordered voices have a less predictable period
sequence → higher PPE). Per cycle `logT = log(period_in_samples)`; sliding
window of 40 cycles; per window subtract the mean, histogram the residual into
**fixed** bins (width 0.01 in log-period, 10 bins over ±0.05), Shannon entropy
normalised by `log(bins)` → PPE ∈ `[0, 1]`.
*Reference:* Little, McSharry, Hunter, Spielman & Ramig (2009), *"Suitability
of dysphonia measurements for telemonitoring of Parkinson's disease"*, IEEE
TBME 56(4).
*Validation:* **PASS** — (C) VOICED corpus: ROC AUC 0.73 (> 0.70), healthy
median 0.282 < pathological 0.333. (Validated despite the `待验证` flag.)
[ppe.md](validation/metrics/ppe.md)

### ZCR — Zero-Crossing Rate (`ZCR`, unitless) — `待验证`
Per-cycle count of sign changes divided by cycle length — a cheap dimensionless
proxy for high-frequency / noise content. Per cycle `[s, e)`:
`crossings = Σ 1[sign(x[k+1]) ≠ sign(x[k])]`, `ZCR = crossings / (e − s)`.
*Reference:* standard signal-processing definition; analytic anchors — white
Gaussian noise → 0.5, a pure sinusoid of frequency `f` (full-signal) → `2f/fs`.
*Validation:* **PASS** — (B) white-noise ZCR → 0.498, ×2 per octave, tone ≪
noise (75×), harmonic monotonicity. (Half-open per-cycle window halves the
absolute scale → relative / ordering metric; see doc §7.)
[zcr.md](validation/metrics/zcr.md)

### GNE — Glottal-to-Noise Excitation Ratio (`GNE`, unitless)
Fraction of the excitation that is glottal (periodic) vs turbulent noise — a
glottal pulse excites all bands synchronously (GNE → 1), breathy phonation
excites them independently (GNE → 0). Native Michaelis algorithm: downsample to
~11 kHz → LPC inverse-filter → per-band analytic (Hilbert) envelopes (1000 Hz
bands centred every 300 Hz over 500–4500 Hz) → for each band pair ≥ bw/2 apart,
the normalised zero-lag envelope cross-correlation; **GNE = max over pairs** per
~10 ms frame, clipped to `[0, 1]`.
*Reference:* Michaelis, Gramss & Strube (1997), *Glottal-to-Noise Excitation
Ratio*. Compared against Praat `To Harmonicity (gne)`.
*Validation:* **PASS** — (B) rises monotonically with SNR (`r = +0.94`, the
defining property), clean 0.91 vs heavy-noise 0.35; (A) positively tracks Praat
GNE (`r = +0.59`). (Replaced a prior anti-correlated proxy, 2026-06-01.)
[gne.md](validation/metrics/gne.md)

### RMS amplitude (`RMS`, unitless) — `待验证`
Time-domain root-mean-square level per frame. `RMS = √(mean(frame²))` on the
shared 25 ms / 10 ms Hann STFT frames, assigned per cycle. A raw level
descriptor (the SPL axis derives from a per-cycle RMS; see SPL below).
*Reference:* standard RMS definition; computed inside `SpectralMomentsCalculator`.
*Validation:* computed; **not yet cross-validated** as a standalone metric
(`待验证`). It is the same RMS primitive validated for SPL
([spl.md](validation/metrics/spl.md)) and shares that doc's framing; no
dedicated acceptance test of its own.

### F0 (`F0_Hz`, Hz) — `待验证`
Fundamental frequency in Hz of each cycle, the linear-frequency form of the VRP
MIDI axis: `F0_Hz = 440·2^((MIDI − 69)/12)`, where MIDI is the NSDF pitch lag
converted via `MIDI = 12·log2(F0/440) + 69`. The pitch itself is the Tartini-
NSDF estimate (see Clarity).
*Reference:* McLeod & Wyvill (2005) for the NSDF pitch; the Hz↔MIDI relation is
the standard equal-temperament conversion.
*Validation:* **PASS** for the underlying pitch — (A) Praat-AC parity (median
F0 error 0.00 %, P90 0.0001 %), (B) NSDF within ±4 cents at 200 / 800 Hz.
F0_Hz is the deterministic Hz transform of that validated pitch (registry-
flagged `待验证` only as a separate column). [f0_clarity.md](validation/metrics/f0_clarity.md)

### Spectral Centroid (`SpectralCentroid`, Hz) — `待验证`
Spectral "centre of mass" / brightness. On the shared 25 ms / 10 ms Hann STFT
(`psd = |rfft|²`, power-weighted): `centroid = Σ psd·f / Σ psd`.
*Reference:* librosa `feature.spectral_centroid` (McFee et al. 2015), validated
on a shared spectrogram so only the formula is under test.
*Validation:* **PASS** — (A) formula == librosa to `max_rel 6.6e-16`; (B) a
1 kHz tone reads centroid 1000.0 Hz through the shipped calculator.
[spectral_moments.md](validation/metrics/spectral_moments.md)

### Spectral Bandwidth (`SpectralBandwidth`, Hz) — `待验证`
Power-weighted spectral spread around the centroid:
`bandwidth = √(Σ psd·(f − centroid)² / Σ psd)`.
*Reference:* librosa `feature.spectral_bandwidth` (McFee et al. 2015).
*Validation:* **PASS** — (A) formula == librosa to `max_rel 4.0e-16` on a shared
spectrogram. [spectral_moments.md](validation/metrics/spectral_moments.md)

### Spectral Rolloff 85 % (`SpectralRolloff85`, Hz) — `待验证`
Frequency below which 85 % of spectral energy lies:
`rolloff85 = min f : cumsum(psd) ≥ 0.85·Σ psd`.
*Reference:* librosa `feature.spectral_rolloff` (McFee et al. 2015).
*Validation:* **PASS** — (A) formula == librosa to `max_rel 0` on a shared
spectrogram. [spectral_moments.md](validation/metrics/spectral_moments.md)

### Spectral Flatness (`SpectralFlatness`, unitless) — `待验证`
Geometric-mean / arithmetic-mean of the magnitude spectrum — 0 for a tonal
spectrum, → 1 for noise: `flatness = geomean(mag) / mean(mag)`.
*Reference:* librosa `feature.spectral_flatness` (McFee et al. 2015).
*Validation:* **PASS** — (A) formula == librosa to `max_rel 0`; (B) white-noise
flatness 0.85 ≫ tone ≈ 0. [spectral_moments.md](validation/metrics/spectral_moments.md)

### Spectral Slope (`SpectralSlope`, unitless) — `待验证`
Overall spectral tilt: the least-squares slope of `log10(|mag|)` vs frequency
over 0–5 kHz (units: `log10` per Hz, ≈ −1e-3…0 for voice — not dB/octave).
*Reference:* analytic ground truth (librosa has no `spectral_slope`); a spectrum
whose `log10|S|` is exactly linear in `f`.
*Validation:* **PASS** — (B) recovers an analytic linear-log-spectrum slope to
`rtol 1e-6`. [spectral_moments.md](validation/metrics/spectral_moments.md)

### Spectral Skewness (`SpectralSkewness`, unitless) — `待验证`
Third (power-weighted) spectral moment about the centroid, normalised by
bandwidth: `skewness = Σ (psd/Σpsd)·(f − centroid)³ / bw_safe³` (bandwidth
floored at 50 Hz, output clipped to ±100).
*Reference:* standard spectral-moment definition; computed inside
`SpectralMomentsCalculator` alongside the librosa-validated moments.
*Validation:* computed; **not yet cross-validated** (`待验证`). The spectral-
moments doc explicitly scopes skewness / kurtosis out of its acceptance as
display-bounded descriptive moments (clipped, bandwidth-floored), so no
numerical parity is asserted yet. [spectral_moments.md](validation/metrics/spectral_moments.md)

### Spectral Kurtosis (`SpectralKurtosis`, unitless) — `待验证`
Fourth (power-weighted) spectral moment about the centroid, minus 3:
`kurtosis = Σ (psd/Σpsd)·(f − centroid)⁴ / bw_safe⁴ − 3` (clipped to −100…1000).
*Reference:* standard spectral-moment definition; computed inside
`SpectralMomentsCalculator`.
*Validation:* computed; **not yet cross-validated** (`待验证`) — same status as
spectral skewness (display-bounded descriptive moment, outside the spectral-
moments acceptance set). [spectral_moments.md](validation/metrics/spectral_moments.md)

### Alpha Ratio (`AlphaRatio`, dB) — `待验证`
Low-vs-high band power ratio (eGeMAPS spectral-balance descriptor of
pressedness / effort): `AlphaRatio = 10·log10( E[50–1000 Hz] / E[1000–5000 Hz] )`.
Note VoiceMap's band order is the **inverse** of OpenSMILE's (sign-flipped vs an
eGeMAPS Alpha threshold).
*Reference:* eGeMAPS (Eyben et al. 2016, IEEE TAC); canonical extractor
OpenSMILE (`eGeMAPSv02`).
*Validation:* **PASS** — (B) two-tone analytic GT exact (≤ 0.03 dB);
(A) `|r| = 1.0` vs OpenSMILE (perfect anti-correlation by the band-order
convention). [alpha_hammarberg.md](validation/metrics/alpha_hammarberg.md)

### Hammarberg Index (`HammarbergIndex`, dB) — `待验证`
Peak-level difference between a low and a high band (eGeMAPS):
`Hammarberg = max(|S|dB over 0–2 kHz) − max(|S|dB over 2–5 kHz)`.
*Reference:* eGeMAPS (Eyben et al. 2016); OpenSMILE `hammarbergIndex_sma3`.
*Validation:* **PASS** — (B) two-tone analytic GT to ≤ 0.03 dB; (A) `r = +0.9999`
vs OpenSMILE, median |Δ| 0.39 dB. [alpha_hammarberg.md](validation/metrics/alpha_hammarberg.md)

### MFCC 1–13 (`mfcc1`…`mfcc13`, unitless) — *CSV-only, not in the metric menu*
The standard timbral feature vector for voice ML — 13 mel-frequency cepstral
coefficients per frame summarising the mel-scaled spectral envelope. Pipeline:
pre-emphasis 0.97 → 25 ms / 10 ms Hamming frames → power spectrum → **26-band
HTK mel filterbank** (`mel = 2595·log10(1 + f/700)`, triangle vertices
quantised to FFT bins) → **natural log** → **DCT-II (ortho)** → first 13
coefficients (C1–C13; no C0, no liftering, no deltas).
*Reference:* librosa `feature.mfcc` (McFee et al. 2015); HTK MFCC (Young et al.,
*HTK Book*). Two deliberate convention choices (vertex-quantised filterbank,
natural-log vs dB) make ours correlation-equal, not bit-equal, to a default
librosa call.
*Note:* MFCCs are intentionally **absent from `metrics_registry.py`** (13
clinically-non-actionable columns would clutter the GUI metric selector) — they
are still computed and written to CSV for power users.
*Validation:* **PASS** — (A) mel centres + DCT-II match librosa exactly; full
MFCC per-coefficient `r ≥ 0.999`. [mfcc.md](validation/metrics/mfcc.md)

## EGG

Electroglottographic descriptors of vocal-fold contact, computed only when a
synchronous EGG channel is present (the GUI greys out this whole category for
mono recordings).

### Qcontact (`Qcontact`, unitless)
VoiceMap's **amplitude-based** contact descriptor (an **identity marker**,
frozen by project rule — *not* the standard time-based contact quotient, which
is `OQ`). Per cycle from the EGG min/max: `Qcontact = cmin / (cmin − cmax)`. It
depends only on the min/max amplitudes, not on contact time, and is negative for
a positive-baseline EGG.
*Reference:* conceptually the Howard (1995) contact-quotient family + the
reference SuperCollider VRP implementation; the shipped amplitude-based formula
is VoiceMap-specific (no external parity source).
*Validation:* **PASS** (implementation spot-check) — `Qcontact == cmin/(cmin −
cmax)` evaluated by hand to `atol 1e-9`; reference / physical parity N/A by
design. [qcontact.md](validation/metrics/qcontact.md)

### dEGGmax (`dEGGmax`, slope)
Peak amplitude of the EGG derivative — a steeper closure (sharper, more abrupt
contact) gives a larger dEGGmax. Per cycle: `amp_sc = 1/(p2p·(−0.5)·sin(2π/T))`
with `p2p = cmin − cmax`; `delta = max(EGG[s] − EGG[s−1], max(diff(cyc)))`;
`dEGGmax = delta · amp_sc`. For an idealised full-amplitude closure this reduces
to the closed form `dEGGmax = 2/sin(2π/T) ≈ T/π`.
*Reference:* VoiceMap / reference SuperCollider VRP convention (paired with
Qcontact / Icontact).
*Validation:* **PASS** — (B) analytic `dEGGmax == 2/sin(2π/T)` to `atol 1e-6`.
[qcontact.md](validation/metrics/qcontact.md)

### Icontact (`Icontact`, unitless)
Index of contacting — a composite of contact speed and contact amount:
`Icontact = log10(max(dEGGmax, 1)) · Qcontact`. Inherits Qcontact's amplitude-
based (identity-marker) sign/scale conventions.
*Reference:* VoiceMap composite definition (`log10(dEGGmax)·Qcontact`).
*Validation:* **PASS** (implementation spot-check) — `Icontact ==
log10(dEGGmax)·Qcontact` to `atol 1e-9`. [qcontact.md](validation/metrics/qcontact.md)

### HRFegg — EGG Harmonic Richness Factor (`HRFegg`, dB)
Energy of the EGG harmonics (H2…HN) relative to the fundamental — richer
harmonics ↔ a sharper, more abrupt contact. Per cycle, from the per-cycle DFT:
`fund = |X1|`, `harms = 2·√(Σ_{k≥2} |Xk|²)`, `HRFegg = 20·log10(harms / fund)`,
with `n_harmonics = 10`.
*Reference:* Howard (1998) — EGG harmonic richness; the reference SuperCollider
VRP implementation's phase-portrait lineage.
*Validation:* **PASS** — (B) recovers the closed form `20·log10(2·√Σ|Xk≥2|²/|X1|)`
to < 1e-3 dB; pure fundamental → ≪ 0 dB. [hrf.md](validation/metrics/hrf.md)

### Open Quotient (`OQ`, unitless)
The **standard time-based** open quotient — the fraction of the cycle the
glottis is open. From the EGG and its derivative: `t_goi = argmin(dEGG)`
(glottal opening instant), `T` = period; `OQ = (T − t_goi) / T`, clipped `[0, 1]`.
*Reference:* Baken & Orlikoff (2000); Howard (1995) — EGG open-quotient
definition (distinct from the amplitude-based `Qcontact`).
*Validation:* **PASS** — (B) synthetic EGG recovers `OQ = 1 − CQ` to ≤ 0.003
over a closed-quotient sweep. [oq.md](validation/metrics/oq.md)

### Speed Quotient (`SPQ`, unitless)
Ratio of glottal opening time to closing time. With `t_peak = argmin(EGG)`
(maximum opening): `SPQ = (t_peak − t_goi) / (T − t_peak)`, clipped 0.1–10.
*Reference:* Baken & Orlikoff (2000); Howard (1995).
*Validation:* **PASS** — (B) synthetic EGG recovers `ramp/(1 − CQ − ramp)` to
≤ 0.013. [oq.md](validation/metrics/oq.md)

### Contact Index (`CIQ`, unitless)
Asymmetry of the contacting vs decontacting phase:
`CIQ = ((T − t_peak) − (t_peak − t_goi)) / (T − t_goi)`, clipped `[−1, 1]`.
*Reference:* Baken & Orlikoff (2000); Howard (1995).
*Validation:* **PASS** — (B) synthetic EGG recovers `(1 − CQ − 2·ramp)/(1 − CQ)`
to ≤ 0.012. [oq.md](validation/metrics/oq.md)

## Singing

Singing-voice descriptors: vibrato, formants, and the singer's-formant /
spectral-ring band measures.

### Vibrato rate (`VibratoRate`, Hz)
Frequency of the periodic F0 modulation of a sustained note (4–8 Hz for trained
singers). Sliding window of `W = 80` cycles over the per-cycle MIDI series; per
window detrend → Hann → **zero-padded** `rfft` (`n_fft = 512`) → magnitude peak
in the `[4, 8] Hz` band → parabolic sub-bin interpolation → **rate** (window
frequency axis `freq[k] = k·F0/n_fft`). Gated to ≥ 80 % voiced cycles and peak
SNR > 2× median.
*Reference:* Sundberg (1995/1987), *The Science of the Singing Voice* (rate
≈ 5–7 Hz for trained voices); validated by synthetic ground truth (FFT-of-MIDI
has a closed-form answer).
*Validation:* **PASS** — (B) imposed 6 / 5 / 7 Hz read 6.00 / 5.01 / 6.99 Hz
(within ±0.3 Hz); a steady note flags vibrato on 0 % of cycles.
[vibrato.md](validation/metrics/vibrato.md)

### Vibrato extent (`VibratoExtent`, cents)
Peak-to-peak depth of the F0 modulation (cents). From the same windowed FFT
peak: `extent = 800 · peak_mag / W` (tuned so a Hann-windowed FFT peak returns
exactly `200·A` cents for an `A`-semitone modulation); gated `≤ 400 cents`.
*Reference:* Sundberg (1995/1987) — extent ≈ ±50–150 cents for trained voices.
*Validation:* **PASS** — (B) formula-level extent within 1–1.4 % of imposed
100 / 50 / 200 cents; ~14 % on real audio (pitch-tracker noise broadens the
peak). [vibrato.md](validation/metrics/vibrato.md)

### F1 / F2 / F3 — formants (`F1`, `F2`, `F3`, Hz)
The three lowest vocal-tract resonances (vowel identity, vocal-tract length,
singer's-formant tuning). A faithful port of Praat `Sound: To Formant (burg)`:
resample to `2·max_formant` → 50 Hz pre-emphasis → Praat Gaussian window → Burg
LPC of order `2·n_formants` → polynomial roots. For each pole `z`:
`freq = |∠z|·fs/2π`; one pole per conjugate pair in `[50, max_formant]`, sorted
ascending → F1, F2, F3. Defaults `n_formants = 5`, `max_formant = 5500 Hz`.
*Reference:* Praat `Sound: To Formant (burg)` (Markel & Gray, *Linear Prediction
of Speech*, 1976). Compared against `parselmouth`.
*Validation:* **PASS** — (A) per-cycle Praat parity on real audio: F1 1.3 %,
F2 2.4 %, F3 0.8 % median |Δf/f| (≤ 5 %); (B) synthetic F1 within 3 %. (Synthetic
F2/F3 confounded by voicing harmonics at high F0 — affects Praat too.)
[formants.md](validation/metrics/formants.md)

### B1 / B2 / B3 — formant bandwidths (`B1`, `B2`, `B3`, Hz) — `待验证`
The −3 dB widths of the first three formants — broad bandwidths indicate
damping / breathiness. The bandwidths of the **same Burg poles** that give
F1/F2/F3: `bw = −ln|z|·fs/π`. Any bandwidth > 800 Hz is zeroed (a width that
large is a mis-located / merged pole; NA-as-0 convention).
*Reference:* Praat `Sound: To Formant (burg)` → `Get bandwidth at time` (the
pole-radius bandwidth of the same Burg LPC). Compared against `parselmouth`.
*Validation:* **PASS** — (A) aggregate-median Praat parity: B1 3.0 %, B2 5.4 %,
B3 1.2 % (medians 182 / 244 / 155 vs Praat 188 / 258 / 157 Hz). High per-cycle
scatter (10–18 %) is intrinsic; only medians are read clinically. (Validated
despite the `待验证` flag.) [bandwidths.md](validation/metrics/bandwidths.md)

### Formant Dispersion (`FormantDispersion`, Hz) — `待验证`
A vocal-tract-length proxy — the average spacing of the lower formants:
`FormantDispersion = (F3 − F1) / 2` when both F1 and F3 are valid (> 0),
otherwise 0. Computed in `FormantExtrasCalculator` from the same Burg poles as
F1/F3.
*Reference:* formant-dispersion as a vocal-tract-length / body-size correlate
(Fitch 1997); derived here directly from the Praat-validated F1/F3.
*Validation:* computed; **not yet cross-validated** as a standalone metric
(`待验证`). It is a deterministic `(F3 − F1)/2` transform of the validated
formants, so its accuracy follows F1/F3 ([formants.md](validation/metrics/formants.md)),
but it has no acceptance test of its own.

### Singer's Formant Energy / SFE (`SingersFormant`, dB)
Energy in the singer's-formant region (~2.8–3.4 kHz, the "ring" of a trained
classical voice) relative to total. Per 25 ms frame (pre-emphasis 0.97,
Hamming), FFT power: `SFE = 10·log10( Σ power[2.8–3.4 kHz] / Σ power )`.
*Reference:* Sundberg (1974) — the singer's formant.
*Validation:* **PASS** — (B) SFE rises monotonically with singer-band energy
(−52 dB with none → ≈ 0 dB when the band dominates). (C) classical-singer-vs-
speech corpus deferred. [singer_formant.md](validation/metrics/singer_formant.md)

### SPR — Singing Power Ratio (`SPR`, dB) — `待验证`
High-band vs low-band energy — another singer's-formant / "ring" indicator:
`SPR = 10·log10( Σ power[2–4 kHz] / Σ power[0–2 kHz] )` on the same Hamming
frames.
*Reference:* Omori et al. (1996) — Singing Power Ratio.
*Validation:* **PASS** — (B) SPR linear in the imposed `20·log10(A_hi/A_lo)`
(slope 0.9996, `r = 1.00`). (C) singer-corpus deferred. (Validated despite the
`待验证` flag.) [singer_formant.md](validation/metrics/singer_formant.md)

### H1-H2 (`H1H2`, dB)
Level of the fundamental minus the 2nd harmonic — a glottal-source / open-
quotient correlate (breathy → large positive H1-H2, pressed → near zero /
negative). Per cycle from the voice DFT: `H1H2 = 20·log10(|X1|/|X2|)`, clipped
±40 dB. Reported **uncorrected** (no vocal-tract / formant correction).
*Reference:* Iseli & Alwan (2004), ICASSP; canonical tool VoiceSauce (Shue et
al. 2011). VoiceSauce additionally applies a formant correction (`H1*-H2*`);
ours is the raw ratio.
*Validation:* **PASS** — (B) recovers `20·log10(A1/A2)` from known harmonics to
max |Δ| 0.046 dB; ±40 dB clip exact. [h1h2.md](validation/metrics/h1h2.md)

### H1-H3 (`H1H3`, dB)
Level of the fundamental minus the 3rd harmonic — the same source descriptor
one harmonic further up: `H1H3 = 20·log10(|X1|/|X3|)`, clipped ±40 dB,
uncorrected.
*Reference:* Iseli & Alwan (2004); VoiceSauce.
*Validation:* **PASS** — (B) recovers `20·log10(A1/A3)` to max |Δ| 0.185 dB
(looser than H1-H2 — the 3rd harmonic is weaker and more leakage-prone).
[h1h2.md](validation/metrics/h1h2.md)

### VibratoJitter (`VibratoJitter`, %) — `待验证`
Coefficient of variation of the vibrato *period* (1/rate) inside a sliding
window — how *steady* the vibrato is (low = even, high = wobbly). Window
`W = min(40, max(n//4, 4))`; per centred window, over cycles with `rate > 0`
form `periods = 1/rate` and emit `100·std(periods)/mean(periods)`.
*Reference:* no external tool; vibrato-rate regularity is a recognised
voice-quality dimension (Sundberg; Dejonckere). Validated against the closed-form
CV of a designed rate sequence.
*Validation:* **PASS** — (B) constant rate → 0 %, alternating period `p0(1±δ)`
→ `CV = 100·δ` exact; end-to-end steady 6 Hz vibrato 0.18 % vs wobbly 7.2 %
(40× separation). (Validated despite the `待验证` flag.)
[vibrato_jitter.md](validation/metrics/vibrato_jitter.md)

## Cluster

Unsupervised per-cell labels grouping cycles by shape or quality. Two parallel
families (EGG-shape clusters and phonation clusters), each with per-cluster
share columns plus a dominant-cluster argmax column. The empty-cluster rescue
(CLAUDE.md §8.7) keeps all five labels populated even when a cluster is sparse.

### EGG-shape clusters (`Cluster 1`…`Cluster 5`, %; `maxCluster`, categorical)
K-means clustering of the EGG cycle **shape** vectors. `Cluster k` is the
percentage of cycles in a cell assigned to EGG-shape cluster `k`; `maxCluster`
is the per-cell `argmax` over the five shares (the dominant shape cluster, drawn
on a 5-colour categorical palette). Normalisation is a z-score
(`sklearn.StandardScaler`); clustering is `KMeans(n_clusters=5, n_init=10,
random_state=0)` with empty-cluster rescue.
*Reference:* scikit-learn (Pedregosa et al. 2011) — `StandardScaler` + `KMeans`.
Validated by the sibling phonation-cluster logic (same pipeline; see cphon.md).
*Validation:* **PASS** — (A) z-score == `StandardScaler` to `atol 1e-9`;
(B) recovers 5 separable blobs at Adjusted Rand 1.0, deterministic under fixed
`random_state`. [cphon.md](validation/metrics/cphon.md)

### Phonation clusters / cPhon (`cPhon 1`…`cPhon 5`, %; `maxCPhon`, categorical)
K-means clustering of the per-cycle **quality-metric profile** into "phonation
types". Features (`DEFAULT_KEYS`, 6 de-correlated representatives): `cpp,
qcontact, spq, jitter, hnr, crest`, stacked to `(n_cycles, 6)`, z-scored, then
`KMeans(n_clusters=5, n_init=10, random_state=0)` with empty-cluster rescue.
`cPhon k` = percentage of a cell's cycles in phonation cluster `k`; `maxCPhon` =
the per-cell dominant phonation cluster (argmax). On mono recordings the
EGG-derived features (`qcontact`, `spq`) drop out, reducing the effective space
to 4.
*Reference:* scikit-learn (Pedregosa et al. 2011). Labels are arbitrary integers
(no ground-truth phonation label exists); `k = 5` is a deliberate granularity
along a continuous quality gradient, not a discovered cluster count (see doc §7).
*Validation:* **PASS** — (A) z-score == `StandardScaler` (Δ 0); (B) recovers
5 separable blobs at Adjusted Rand 1.0; deterministic. The 6-feature set
(retired from a redundant 9) balances the clusters so all 5 `maxCPhon` labels
appear. [cphon.md](validation/metrics/cphon.md)

## Density

### Total (`Total`, cycles)
The number of analysed cycles in each `(MIDI, dB)` cell — the cycle-count layer
of the VRP (aggregated by `sum`, displayed on a log colour scale). Not a
measured acoustic quantity but the occupancy / dwell-time map underlying the
profile; it is what every per-cell aggregate is computed over. (Density =
`Total` only — the former whole-recording broadcast scalars MPT / VoicingRatio /
DUV were removed as having no per-cell meaning.)
*Reference:* the Voice Range Profile occupancy convention (Schutte & Seidner
1983; the reference SuperCollider VRP implementation).
*Validation:* a deterministic per-cell count (no algorithmic ambiguity);
exercised by the end-to-end CSV regression (`scripts/e2e_regression.py`, 0
drift across the three recording modes) rather than a dedicated metric doc.
