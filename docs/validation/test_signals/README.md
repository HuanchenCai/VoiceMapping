# Synthetic Test Signals

Generated by [`make_signals.py`](make_signals.py). Run:

```bash
python docs/validation/test_signals/make_signals.py
```

This writes 12 mono float32 WAVs (44.1 kHz) + [`manifest.json`](manifest.json),
each with analytically-known ground truth. Regenerating is deterministic
(fixed seeds), so the files are reproducible and need not be committed as
binaries — CI regenerates them before validating.

## Synthesis model

Classic source-filter:

```
Rosenberg glottal flow (continuous-phase, per-cycle period + amplitude)
  → Klatt formant cascade (2-pole resonators, unity DC gain)
  → lip radiation (first difference)
  → [+ optional Gaussian noise for breathy / SNR control]
```

Period (jitter) and amplitude (shimmer) are imposed as a strict
alternating ±d pattern, which makes the Praat **local** definition exact:

| imposed | relation | recovered by formula from true marks |
|---|---|---|
| `jitter_local = 2·d_p` | `period_k = T₀(1 ± d_p)` | exact (≤1e-5 rel) |
| `shimmer_local = 2·d_a` | `amp_k = A₀(1 ± d_a)` | exact at amplitude-tier layer |

**Continuous-phase rendering matters**: a 0.5 % jitter at 200 Hz is a
±0.55-sample period change. Rendering pulses at integer-sample onsets
would round that away (and inject spurious jitter into the *modal*
signal). We instead evaluate the glottal pulse from a continuous phase
function sampled at the output rate, so sub-sample period changes survive.

## Important: what synthetic jitter/shimmer can and cannot validate

A perturbation metric is **cycle-marker + formula**. These signals
validate the **formula** layer against ground truth (feed the known
cycle marks → recover the imposed value exactly). They do **not**
validate end-to-end recovery, because Praat's `PointProcess (cc)`
cycle-marker smooths sub-sample period alternation toward a regular grid
(e.g. an imposed 2 % alternating jitter is detected as ~0.22 % once the
marker re-locks the pulses). That marker behaviour is a property of the
F0 / cycle-tracking stage (validated in `metrics/f0_clarity.md`), not of
the jitter formula. See `metrics/jitter.md` §7.
