# Synthetic Test Corpus (public release)

The **synthetic corpus** is VoiceMap's releasable ground-truth dataset and the
primary asset behind the "(B) synthetic ground truth" validation evidence
(`docs/methodology.md`). Unlike the real-voice corpora (`voiced.md`,
`saarbruecken.md`), it can be redistributed freely because it contains **no
recorded human voice** — every signal is produced by source-filter synthesis
with an analytically-known answer.

## Contents

12 mono 44.1 kHz float WAVs + `manifest.json`, generated by
`docs/validation/test_signals/make_signals.py` (see that folder's
[`README.md`](../test_signals/README.md) for the full signal list and the
synthesis model). Ground truth per signal — F0, jitter, shimmer, formants, SNR,
vibrato — lives in `manifest.json` under each signal's `ground_truth` key.

Because generation is deterministic (fixed seeds), the WAVs are **not committed
as binaries**; they are regenerated on demand (CI regenerates them before every
validation run).

## Packaging a release

```bash
python scripts/package_corpus.py
# -> result/voicemap_synthetic_corpus_v<version>.zip
#    (12 WAVs + manifest.json + README.md + LICENSE.txt + CITATION.txt)
```

The script regenerates the signals, bundles them with a license + citation, and
prints the `gh release create …` command to attach the archive to a GitHub
release. The corpus is released under **CC0 1.0** (public-domain test data); the
generator software is MIT.

## Why synthetic for the public corpus

- **Redistributable** — no consent / PII / licensing issues that block sharing
  real clinical recordings.
- **Exact ground truth** — the imposed F0 / jitter / shimmer / formants are
  known to machine precision, so a downstream user can verify VoiceMap (or any
  tool) reproduces the documented values.
- **Reproducible** — anyone can regenerate the identical corpus from source.

Real-voice corpora (Saarbrücken, VOICED) are used internally for the "(C)
real-corpus behaviour" evidence where licensing permits, but are **not**
redistributed here; see their per-corpus docs for access terms.
