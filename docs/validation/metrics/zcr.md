# ZCR — Zero-Crossing Rate

> Per-cycle count of sign changes divided by the cycle length (in samples).
> A cheap, dimensionless proxy for the high-frequency / noise content of the
> waveform: low for a clean low tone, high for noisy / fricated / breathy
> phonation.

## 1. Implementation
- File: `voicemap/metrics.py` → `ZCRCalculator` (lines 2385–2408).
- Per cycle `[s, e)`: `crossings = Σ 1[sign(x[k+1]) ≠ sign(x[k])]`, then
  `zcr = crossings / (e − s)`. Cycles shorter than 2 samples → 0.
- Registry key `ZCR` (Acoustic, `待验证=True`).

## 2. Reference Standard
- Standard signal-processing definition: the zero-crossing rate is the rate at
  which a signal changes sign. For a band-limited signal the *conventional*
  full-signal ZCR (crossings per sample) of a pure sinusoid of frequency `f` is
  `2·f / fs` (two crossings per period). White Gaussian noise has the
  analytic ZCR `0.5` — two consecutive i.i.d. zero-mean samples have opposite
  signs with probability exactly ½, independent of the window.
- No external tool gives a stable per-cycle parity (the count is alignment-
  sensitive, see §7), so the analytic white-noise anchor and the
  frequency-scaling law are the reference.

  1. white-noise ZCR `≈ 0.5` (tol `0.05`) — window-independent analytic value;
  2. ZCR scales linearly with the dominant frequency — an octave up doubles it

  3. a pure tone reads `≪` noise (≥ 10×);
  4. higher harmonics raise ZCR (`pure < +3rd < +5th`);
  5. silence → no crossings.

## 7. Known Limitations
- **Half-open-window halving (absolute scale).** Each cycle window `[s, e)` is
  half-open and starts near a zero crossing, so it captures only one of a pure
  tone's two per-period crossings. The per-cycle ZCR of a clean tone is
  therefore ≈ `f0 / fs` — roughly **½** the conventional full-signal `2·f0/fs`.
  The metric is meaningful as a *relative* high-frequency / noisiness index and
  for *ordering* (tone < harmonics < noise), **not** as an absolute crossing
  count. librosa's `zero_crossing_rate` is equally alignment-sensitive (it
  reports 1 crossing/period at 100 Hz but 2 at 200/400 Hz on the same tones),
  so it is not a stable parity reference.
- **Amplitude/phase dependence for multi-harmonic signals.** The crossing
  count of a sum of sinusoids depends on their relative amplitudes and phases,
  not just the highest harmonic, so only the pure-tone scaling is analytic.
- **No DC/own-mean removal.** Crossings are counted around 0, not around the
  per-cycle mean; a large DC offset suppresses crossings. The voice channel is
  ~zero-mean so this is immaterial in practice.
- **(C) corpus distribution** (typical breathy vs modal ranges) deferred.
