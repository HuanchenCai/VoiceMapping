# PPE (Pitch Period Entropy)

> Entropy of the cycle-to-cycle pitch-period variation after de-trending.
> A nonlinear dysphonia marker (Little et al. 2009) — disordered voices have
> a *less predictable* period sequence, hence higher PPE. The first metric in
> this framework validated by **(C) real-corpus** evidence.

## 1. Implementation
- File: `voicemap/metrics.py` → `PPECalculator` (lines 2290–2358).
- Per cycle: `logT = log(period_in_samples)`; sliding window of 40 cycles;
  per window: subtract the mean, histogram the residual into **fixed** bins
  (width 0.01 in log-period, 10 bins spanning ±0.05 ≈ ±5 % jitter, outliers
  clipped to edge bins), Shannon entropy normalised by `log(bins)` → PPE ∈
  `[0, 1]`. Edge cycles padded with 0.
- The **fixed** (not per-window) bin range is the key fix (commit `f208fce`):
  per-window ranges saturate PPE near 1 for stable phonation; fixed bins make
  it track period *irregularity* magnitude.

## 2. Reference Standard
- **Little, McSharry, Hunter, Spielman & Ramig (2009)**, *"Suitability of
  dysphonia measurements for telemonitoring of Parkinson's disease"*,
  IEEE TBME 56(4) — PPE definition and its discriminative use.
- PPE has no single numeric "reference value" (it depends on the windowing and
  binning); it is validated by its **discrimination**, i.e. healthy < disordered
  on a labelled corpus.

## 3. Test Signals / Corpus
- **VOICED** (PhysioNet; Cesari et al. 2018) — 208 subjects, sustained /a/,
  8 kHz, per-subject diagnosis (healthy vs four dysphonia families),
  ODC-BY 1.0. Fetched by `scripts/fetch_voiced_corpus.py` into
  `docs/validation/corpora/voiced/{healthy,pathological}/*.wav`
  (gitignored) + `manifest.json`.
- PPE is **sample-rate-independent** (it works on log-period *ratios*), so the
  8 kHz recordings are used as-is.

## 4. Validation Method
- **(C) Real-corpus discrimination.** Compute median PPE per recording (cycle
  marks from Praat cc — validated equivalent to VoiceMap's own marker in
  `f0_clarity.md`), split by label, and require:
  - **ROC AUC (pathological > healthy) ≥ 0.70**;
  - pathological median PPE > healthy median (correct direction);
  - ≥ 20 recordings per class.
- **(A)/(B)** — not applicable: PPE is a bounded normalised entropy with no
  external parity tool and no closed-form synthetic target; the corpus test is
  the meaningful one.

Reproducible steps:
```bash
python scripts/fetch_voiced_corpus.py        # download the corpus (once)
python scripts/validate_metric.py ppe
```

## 7. Known Limitations
- **Discriminator, not an absolute scale.** A VoiceMap PPE value (~0.28 healthy
  / ~0.33 disordered on VOICED) is **not** comparable to Little 2009's absolute
  numbers — the windowing and fixed-bin width differ. Only the *ordering* and
  the AUC are validated. Re-derive any clinical cut-off on this scale.
- **AUC depends on the downloaded subset.** The fetch script drops flaky
  PhysioNet downloads, so the exact AUC moves slightly run-to-run (0.68 on
  80 recordings → 0.73 on 92). It is comfortably > 0.70 with the full balanced
  set; a single small download may dip below. Re-fetch for a fuller cohort.
- **Single-feature ceiling.** PPE alone is a moderate discriminator (AUC ~0.73);
  Little 2009 combined it with other measures for high accuracy. VoiceMap
  exposes it as one feature among many — not a standalone diagnostic.
- **Not CI-gated.** The corpus is gitignored (too large), so the harness SKIPS
  this metric when the corpus is absent (CI). It is a local/manual check.
