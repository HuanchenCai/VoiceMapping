# Crest factor

> Peak-to-RMS ratio of the waveform within each glottal cycle. A simple
> "peakiness" measure — pressed/clear phonation is peakier (higher crest),
> breathy/noisy is flatter.

## 1. Implementation
- File: `voicemap/metrics.py` → `CrestCalculator` (lines 531–554).
- Per cycle `[s, e)`: `crest = max(|x|) / RMS(x)`. The voice is delayed by
  20 ms first (`delay = 0.02·sr`) to align the acoustic cycle with its EGG
  trigger; for a steady waveform this shift does not change the per-cycle
  crest. Cycles shorter than `min_samples` → 0.

## 2. Reference Standard
- Standard definition: crest factor `= peak / RMS` (Baken & Orlikoff 2000;
  general signal processing). It is an **analytic constant per waveform
  shape**, independent of amplitude or frequency:
  - sine → `√2 ≈ 1.4142`;
  - square → `1`;
  - sawtooth (ramp) → `√3 ≈ 1.7321`.
- No external tool needed — the closed-form constants are the reference.

## 7. Known Limitations
- **~0.1 % residual** from non-integer samples-per-cycle and the 20 ms
  voice-delay shift (a cycle window does not start exactly at a sample that is
  a waveform zero/peak). Negligible.
- **Sensitive to clipping / spikes.** A single sample spike inflates the peak
  and hence the crest; the metric assumes a clean per-cycle waveform. Real
  recordings with transient clicks read high — gate on Clarity/SPL.
- The 20 ms delay is tuned to the voice↔EGG acoustic lag; on a mono (no-EGG)
  recording it simply shifts the analysis window and is immaterial for steady
  phonation.
