# GNE — Glottal-to-Noise Excitation ❌ (mislabeled — see §6/§7)

> **The shipped `GNE` column does NOT measure Glottal-to-Noise Excitation.**
> It is a `[500–1500] / [1500–2500] Hz` band-energy *symmetry* ratio, tagged in
> the source as "GNE-like (simplified)". Validation against the real Praat GNE
> shows it is **strongly anti-correlated (r ≈ −0.98)** — it points the wrong
> way. Kept frozen pre-copyright; flagged for a post-freeze rename/replace.

## 1. Implementation
- File: `voicemap/metrics.py` → `FormantExtrasCalculator.calculate`
  (lines 1539–1545), output key `gne`.
- `e_g1 = Σ PSD[500–1500 Hz]`, `e_g2 = Σ PSD[1500–2500 Hz]` on 25 ms /
  10 ms pre-emphasised (0.97) Hamming frames; `gne_proxy = min(e_g1,e_g2) /
  max(e_g1,e_g2)` ∈ [0, 1]. Assigned to each cycle by frame index.
- Registry key `GNE` (Acoustic, `待验证=True`, label "GNE-like (simplified)").

## 2. Reference Standard
- **Real GNE** — Michaelis, Gramss & Strube (1997), *Glottal-to-Noise
  Excitation Ratio*: inverse-filter to the excitation, take Hilbert envelopes
  in overlapping frequency bands, and compute the **maximum cross-correlation**
  between band pairs. A glottal pulse excites all bands synchronously
  (correlated envelopes → GNE → 1); turbulent noise excites them independently
  (uncorrelated → GNE low). High = clean/glottal, low = noisy/breathy.
- **Reference implementation: Praat** `Sound: To Harmonicity (gne)` (bands
  500–4500 Hz, bandwidth 1000 Hz, step 80 Hz) → `Get maximum`. Exposed through
  parselmouth, so a faithful GNE is available as a drop-in replacement.

## 3. Test Signals
- Synthetic breathy→clean vowels: `synth_vowel(200 Hz, neutral)` at
  SNR = 0/5/10/15/20/25 dB + clean — the noise axis GNE is built to measure.
- Two-tone (1000 Hz in band g1 + 2000 Hz in band g2) amplitude sweep — to
  characterise what the proxy actually responds to.

## 4. Validation Method
- **(A) Parity / correlation vs Praat GNE** over the SNR sweep:
  - the reference itself must behave (Praat GNE rises with SNR, `r ≥ +0.70`);
  - **verdict** — to be GNE the proxy must *positively* track Praat GNE
    (`r ≥ +0.50`). **It does not** (`r ≈ −0.98`) → this row FAILS by design.
  - the anti-correlation is quantified (`r ≤ −0.90`).
- **(B) Characterisation** — a two-tone sweep shows the proxy is a `[0,1]`
  band-symmetry ratio (balanced bands → ~1, one band dominant → ~0).

## 7. Known Limitations
- **It is not GNE, and it points the wrong way.** Over a breathy→clean SNR
  sweep the proxy falls toward 0 for clean voice while the real Praat GNE rises
  toward 1 (anti-correlation r ≈ −0.98). Unlike CPP — which differs from Praat
  CPPS in absolute scale but *preserves order* (r = +0.98, re-calibratable) —
  this proxy reverses the order, so it cannot be salvaged by re-scaling, only
  by negation, and even then it is a weaker noise indicator than the real GNE.
- **Near-zero on normal voice.** Clean voiced vowels have very asymmetric
  500–1500 vs 1500–2500 Hz band energy, so the proxy reads ≈ 0 almost
  everywhere (e.g. 0.0017 on a clean 200 Hz vowel) — a nearly flat VRP column.
- **Pre-emphasis shifts the "balanced" point.** The 0.97 pre-emphasis boosts
  the upper band, so the symmetry peak sits at amplitude ratio ≈ 0.5, not 1.0
  (it is a band ratio of pre-emphasised energies, not raw ones).
- **Recommended post-freeze action.** Either (a) rename the column to a
  band-symmetry / spectral-balance descriptor to stop implying GNE, or
  (b) replace it with a real GNE — Praat's `To Harmonicity (gne)` is available
  through parselmouth and is a faithful drop-in. Tracked in PLAN §6.
