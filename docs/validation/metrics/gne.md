# GNE — Glottal-to-Noise Excitation Ratio

> Measures how much of the excitation is glottal (periodic) vs turbulent
> noise. A glottal pulse excites all frequency bands synchronously, so their
> envelopes are correlated (GNE → 1); breathy / noisy phonation excites bands
> independently (GNE → 0). High = clean glottal source, low = noisy / breathy.

> **History:** the column previously shipped a `min/max` band-energy *symmetry*
> ratio mislabeled as GNE — it was **anti-correlated (r ≈ −0.98)** with the
> real GNE. It was replaced (2026-06-01) with a native Michaelis (1997)
> implementation; this doc validates the replacement.

## 1. Implementation
- File: `voicemap/metrics.py` → `_compute_gne` (module-level helper) +
  `FormantExtrasCalculator.calculate` (assigns it per cycle, output key `gne`).
- Algorithm (Michaelis, Gramss & Strube 1997), native — no parselmouth:
  1. downsample to ~11 kHz (cheap IIR `decimate`);
  2. LPC inverse-filter → excitation (autocorr method, needed lags only);
  3. per-band analytic (Hilbert) envelopes via FFT band-masking, decimated —
     bands of 1000 Hz centred every 300 Hz over 500–4500 Hz;
  4. for each band pair ≥ bw/2 apart, the normalised zero-lag cross-correlation
     of envelopes; **GNE = max over pairs**, per ~10 ms frame, clipped to [0,1].
- Registry key `GNE` (Acoustic). ~3 s for a 70 s recording.

## 2. Reference Standard
- Michaelis, Gramss & Strube (1997), *Glottal-to-Noise Excitation Ratio*.
- **Reference implementation: Praat** `Sound: To Harmonicity (gne)`
  (500–4500 Hz, bandwidth 1000 Hz, step 80 Hz) → `Get maximum`, via parselmouth.
- The physical truth GNE encodes: it **decreases monotonically with added
  noise** (this is what it exists to measure).

## 3. Test Signals
- Synthetic breathy→clean vowels `synth_vowel(200 Hz, neutral)` at
  SNR = 0/5/10/15/20/25 dB + clean — the noise axis GNE is built to measure.

## 4. Validation Method
- **(B) Synthetic ground truth (defining property)** — GNE must rise
  monotonically with SNR over the sweep (`corr(GNE, SNR) ≥ 0.90`); clean voice
  reads high (`≥ 0.80`) and `clean ≥ 2× heavy-noise`; output bounded to `[0,1]`.
- **(A) Correlation vs Praat GNE** — positively tracks `To Harmonicity (gne)`
  (`r ≥ 0.50`); the reference itself rises with SNR (`r ≥ 0.70`).

Reproducible steps:

python scripts/validate_metric.py gne

## 7. Known Limitations
- **Moderate Praat parity (r ≈ +0.59), strong noise-ordering (r ≈ +0.94).**
  Ours uses sparser bands (step 300 Hz vs Praat's 80 Hz) and zero-lag envelope
  correlation on an ~11 kHz downsample, so the *absolute* curve sits below
  Praat's at low SNR (ours 0.35 vs Praat 0.75 at SNR 0). The clinically-relevant
  property — the monotonic noise ordering — is strong. Denser bands + a lag
  search would tighten the Praat parity at a compute cost; deferred.
- **Global inverse filter.** A single LPC inverse filter is applied over the
  whole recording (not per-frame), a speed/accuracy trade-off; adequate for the
  spectral flattening GNE needs.
- **(C) corpus distribution** (healthy vs pathological GNE ranges) deferred —
  needs a disordered-voice corpus (e.g. SVD).
