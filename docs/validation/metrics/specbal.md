# SpecBal — spectral balance (high/low band level, dB)

> The dB difference between a high band and a low band of the voice spectrum.
> A simple spectral-tilt / "brightness" measure related to vocal effort and
> mode — pressed/bright phonation raises the high band.

## 1. Implementation
- File: `voicemap/metrics.py` → `SpecBalCalculator` (lines 492–525), with
  `_blp4_sos` / `_bhp4_sos` (lines 472–490) building 4th-order Butterworth-
  style LP/HP cascades (SC `BLowPass4` / `BHiPass4` equivalents, `rq=1.4`).
- Low band = `BLowPass4 @ 1500 Hz`, high band = `BHiPass4 @ 2000 Hz`; per band
  a sliding RMS → dB; `SpecBal = hi_dB − lo_dB`, assigned per cycle, clipped to
  ±50 dB.

## 2. Reference Standard
- The project's SuperCollider source (`BLowPass4` / `BHiPass4` + RMS dB) — the
  metric is an SC port. No external Python tool computes this exact band split,
  so validation is by **synthetic ground truth** on a controlled two-tone.
- Analytic anchor: a low tone (in the LP passband) + a high tone (in the HP
  passband) at amplitude ratio `A_hi/A_lo` gives
  `SpecBal = 20·log10(A_hi/A_lo)` up to a constant filter-passband offset.

## 3. Test Signals
Synthetic two-tone (in-harness): 300 Hz (LP passband) + 4000 Hz (HP passband),
amplitude ratios `A_hi/A_lo ∈ {0.25, 0.5, 1, 2, 4}`, F0-trigger grid at 150 Hz.

## 4. Validation Method
- **(B) Synthetic ground truth** — regress SpecBal on the imposed
  `20·log10(A_hi/A_lo)`:
  - **slope ∈ [0.98, 1.02]** (one dB of band imbalance → one dB of SpecBal);
  - **Pearson r ≥ 0.99**;
  - constant offset `|c| ≤ 1 dB` (the filter-passband-gain bias).
- **(A)** SC-source parity unavailable (no Python reference for the exact band
  split). **(C)** corpus deferred.

Reproducible steps:
```bash
python scripts/validate_metric.py specbal
```

## 7. Known Limitations
- **Constant ~0.4 dB offset** from the 4th-order LP/HP passband gains at the
  test tones — a fixed bias, not a scale error. Absolute SpecBal is therefore
  filter-design-dependent; the *change* in SpecBal (the slope-1 response) is
  what is intrinsically validated. Re-derive any absolute SpecBal cut-off on
  VoiceMap's filters, not on an idealised band split.
- **Crossover gap 1500–2000 Hz.** Energy between the LP cutoff (1500) and the
  HP cutoff (2000) is attenuated in *both* bands, so SpecBal is insensitive to
  spectral changes in that 500 Hz notch. Intentional (separates the bands), but
  worth knowing.
- **No SC numerical parity.** The `rq=1.4` was tuned to match SC's
  `BLowPass4/BHiPass4` empirically; without the SC reference running, only the
  synthetic linearity is checked, not bit-parity with the SC UGen.
