# SPL — Sound Pressure Level (per cycle)

> Per-cycle level in dB. The vertical axis of the Voice Range Profile. Raw
> output is `20·log10(RMS)` (dB relative to digital full scale, negative); a
> `+120 dB` calibration offset converts it to absolute dB SPL.

## 1. Implementation
- File: `voicemap/metrics.py` → `SPLCalculator` (lines 130–155). Per cycle
  `[s, e)`: `RMS = √(mean(x²))` (vectorised cumsum), `SPL = 20·log10(RMS)`;
  cycles shorter than `min_samples` → −100 dB.
- Calibration: `voicemap/csv_writer.py` adds `config.spl_correction_db`
  (= 120.0) at export → absolute dB SPL.

## 2. Reference Standard
- **IEC 61672** (sound level meters) for the dB law; the digital→SPL
  calibration follows the project's SuperCollider `scserver` convention
  (digital full scale ↔ 120 dB SPL).
- Analytic anchor: a sine of amplitude `A` has `RMS = A/√2`, so
  `SPL = 20·log10(A/√2)`, and every amplitude doubling is exactly
  `+20·log10(2) = +6.0206 dB`.

## 7. Known Limitations
- **Absolute SPL is calibration-dependent.** The `+120 dB` offset assumes the
  SC full-scale = 120 dB SPL convention; on a different recording chain (mic
  gain, interface) the *absolute* dB SPL is offset. Only the *relative* level
  (and the dB law) is intrinsically validated — re-calibrate `spl_correction_db`
  per recording setup for clinically-absolute SPL.
- **RMS is unweighted** (no A/C-weighting); it is a broadband level, not a
  perceptual loudness. Appropriate for the VRP axis, not for a weighted SPL.
- **Per-cycle RMS** assumes the cycle boundaries are correct; at voicing
  onsets/offsets a mis-marked cycle gives a transient SPL. Gate on Clarity.
