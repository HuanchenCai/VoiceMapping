# cPhon — phonation-type clustering (K-means)

> An unsupervised label (1..k) per cycle, grouping cycles by their
> quality-metric profile into "phonation types". Not a measured quantity — a
> clustering of the other metrics — so it is validated as an **ML pipeline**.

## 1. Implementation
- File: `voicemap/metrics.py` → `PhonClusterCalculator` (lines 2646–2730+).
- Features: per-cycle `clarity, cpp, specbal, crest, entropy, qcontact,
  deggmax, icontact, hrf` stacked → `(n_cycles, 9)`; non-finite rows dropped.
- Normalisation: **z-score** `z = (x − mean) / std` (population std, floored at
  1e-12) — i.e. `sklearn.preprocessing.StandardScaler`.
- Clustering: `sklearn.cluster.KMeans(n_clusters=5, n_init=10,
  random_state=0)`, with an **empty-cluster rescue** (reassign the worst-fit
  point so every k is populated — see CLAUDE.md §8.7).
- A sibling `ClusterCalculator` (line 2514) clusters EGG cycle *shapes*; the
  same sklearn-standard validation logic applies.

## 2. Reference Standard
- **scikit-learn** (Pedregosa et al. 2011) — `StandardScaler` + `KMeans` are
  the de-facto standard. There is no "ground-truth phonation label" on real
  voice, so the validatable claims are: the normalisation is exactly a
  z-score, and the clustering recovers a *known* grouping on separable data.

## 3. Test Signals
Synthetic feature matrices (in-harness, seeded): random vectors for the
normalisation check; 5 well-separated Gaussian blobs in 9-D (60 points each)
with known labels for the recovery check.

## 4. Validation Method
- **(A) Normalisation parity** — `(x − μ)/σ` matches
  `StandardScaler().fit_transform` (`atol = 1e-9`).
- **(B) Cluster recovery + determinism** — on 5 separable blobs the pipeline's
  labels match the ground-truth grouping (**Adjusted Rand ≥ 0.99**), and two
  runs with the same `random_state` give identical labels.
- **(C)** no real-corpus label to validate against (cPhon is unsupervised).

Reproducible steps:
```bash
python scripts/validate_metric.py cphon
```

## 7. Known Limitations
- **Labels are arbitrary integers**, not ordered phonation types — cluster `2`
  has no fixed meaning across recordings unless centroids are aligned to a
  reference. The *grouping* is validated, not a semantic label.
- **Empty-cluster rescue perturbs the pure K-means** result (one worst-fit
  point is force-moved). Deliberate (keeps all k columns in the VRP), but the
  rescued solution is not the raw sklearn optimum. Documented; do not "simplify"
  it away (CLAUDE.md §8.7).
- **Centroid physical meaning is a research question** — whether cluster k
  corresponds to "pressed / modal / breathy" is out of P1 scope (frozen).
- **Feature set is fixed** to the 9 DEFAULT_KEYS; changing it changes the
  clustering. EGG-derived features (qcontact, deggmax, icontact, hrf) are 0 on
  mono recordings, reducing the effective feature space.
