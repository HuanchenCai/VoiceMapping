# Alpha Ratio + Hammarberg Index

> Two spectral-balance descriptors from the eGeMAPS affect/voice-quality set.
> Both quantify how energy is split between a low and a high band — markers of
> phonation pressedness / breathiness and vocal effort.

## 1. Implementation
- File: `voicemap/metrics.py` → `SpectralMomentsCalculator` (lines 797–810).
- `AlphaRatio = 10·log10( E[50–1000 Hz] / E[1000–5000 Hz] )` (power ratio).
- `Hammarberg = max(|S|dB over 0–2 kHz) − max(|S|dB over 2–5 kHz)` (peak-level
  difference).
- Same 25 ms / 10 ms Hann STFT as the other spectral moments; per-cycle
  assignment.

## 2. Reference Standard
- **eGeMAPS** (Eyben et al. 2016, *"The Geneva Minimalistic Acoustic Parameter
  Set…"*, IEEE TAC); the canonical extractor is **OpenSMILE**
  (`eGeMAPSv02`, LLDs `alphaRatio_sma3`, `hammarbergIndex_sma3`).
- **Convention difference:** OpenSMILE's AlphaRatio is the **inverse** band
  ratio (high/low), so ours and OpenSMILE are **sign-flipped** for Alpha (a
  perfect anti-correlation); Hammarberg shares the same sign.
- **Analytic anchor:** a two-tone signal `A_low·sin(2π·500t) +
  A_high·sin(2π·3000t)` has, by construction,
  `AlphaRatio = Hammarberg = 20·log10(A_low/A_high)` exactly.

## 3. Test Signals
- Two-tone pairs (in-harness), `A_low/A_high ∈ {0.25, 0.5, 1, 2, 4}` — analytic
  GT.
- A voiced vowel (`synth_vowel`, F0 = 200) with a one-pole spectral-tilt sweep
  (`a ∈ {−0.97 … 0.97}`) — the OpenSMILE comparison set (OpenSMILE only
  produces meaningful Alpha/Hammarberg on *voiced* input).

## 4. Validation Method
- **(B) Analytic ground truth:** Alpha and Hammarberg each recover
  `20·log10(A_low/A_high)` to **≤ 0.5 dB** over the 5 two-tone ratios.
- **(A) OpenSMILE eGeMAPS parity** on the voiced tilt sweep:
  - Alpha: `|Pearson r| ≥ 0.95` (it is ≈ −1.0 — sign-flipped by convention);
  - Hammarberg: `r ≥ 0.95` **and** median `|Δ| ≤ 2 dB` (near value-parity).
- **(C) Real-corpus behaviour** — deferred to the corpus phase.

## 7. Known Limitations
- **Alpha sign is opposite to OpenSMILE eGeMAPS.** Ours = `E_low/E_high`,
  OpenSMILE = `E_high/E_low`; the magnitudes agree but a VoiceMap Alpha must
  be *negated* before comparing to an eGeMAPS Alpha threshold. Documented, not
  changed (frozen; flipping it would move the 48-PASS baseline).
- **Band edges are fixed** (`[50,1000]`/`[1000,5000]` for Alpha,
  `[0,2k]`/`[2k,5k]` for Hammarberg). eGeMAPS uses the same nominal bands, but
  OpenSMILE adds voicing gating + spectral smoothing, so per-frame values can
  differ on unvoiced/transition frames even though voiced-region medians match
  to < 0.5 dB.
- **OpenSMILE needs voiced input.** On a non-voiced synthetic (e.g. a bare
  two-tone) OpenSMILE's F0-gated extractor returns erratic Alpha/Hammarberg, so
  the OpenSMILE comparison is run on voiced vowels only; the two-tone is used
  solely for the analytic (ours-only) GT.
