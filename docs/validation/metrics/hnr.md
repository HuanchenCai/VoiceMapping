# HNR (Harmonics-to-Noise Ratio) + NHR

> Ratio of periodic (harmonic) to aperiodic (noise) energy in the voice.
> The headline acoustic measure of "breathiness / hoarseness". HNR is in
> **dB**; NHR is the linear noise-to-harmonics ratio (clinical, MDVP-style).

## 1. Implementation
- File: `voicemap/metrics.py`
  - `HNRCalculator` — lines 2404–2483. Praat-style autocorrelation HNR on
    Hann-windowed 40 ms frames, 10 ms hop, pitch search `[f_min, f_max] =
    [60, 400] Hz`. Per frame: `p = max_τ r(τ)` (window-autocorrelation
    compensated, `r(0)`-normalised); `HNR = 10·log10(p/(1-p))`, with the
    peak clipped to `[1e-6, 0.9999]` (a **40 dB ceiling**). Each cycle is
    assigned its enclosing frame's HNR.
  - `NHRCalculator` — lines 2234–2249. `NHR = 1 / H` where `H` is the
    *linear* HNR (`H = 10^(HNR_dB/10)`); a thin inverse of the per-cycle HNR.
- Window-autocorrelation compensation (`acf / win_acf`) is essential —
  without it HNR is ~10–15 dB low (Praat applies the same correction).

## 2. Reference Standard
- **Praat** (Boersma & Weenink), `To Harmonicity (cc)` /
  `Sound_to_Harmonicity_cc`; Boersma (1993), *"Accurate short-term analysis
  of the fundamental frequency and the harmonics-to-noise ratio of a sampled
  sound"*, IFA Proceedings 17, 97–110. Compared against `parselmouth`
  `to_harmonicity_cc(time_step=0.01, minimum_pitch=60, silence_threshold=0.1,
  periods_per_window=4.5)`.
- **Physical anchor** (the rigorous bar): for a periodic signal plus additive
  white noise at signal-to-noise ratio `S` (dB), the harmonic fraction
  `p = E_h/(E_h+E_n)` gives

  ```
  p / (1 - p) = E_h / E_n = 10^(S/10)
  HNR_dB      = 10·log10(p/(1-p)) = S        (exactly)
  ```

  so a correctly-implemented HNR must read back the imposed SNR.
- NHR: MDVP Noise-to-Harmonics Ratio; `NHR = 1/H`. Pathology cut ≈ 0.19
  (≈ HNR 7.2 dB) — Baken & Orlikoff (2000).

  - clean modal HNR ≥ 30 dB (capped at 40).
- **(C) Real-corpus behaviour** — deferred to the corpus phase; see §7 for

## 7. Known Limitations
- **Non-stationary divergence from Praat (~6 dB).** On real connected voice
  (`test_Voice_EGG`, 10 s) ours reads **24.4 dB** vs Praat **17.7 dB**
  (|Δ| ≈ 6.7 dB), even though on stationary vowels they agree to < 0.3 dB.
  Cause: different analysis window (a fixed **40 ms** vs Praat's **4.5
  periods**), frame rate, and aggregation (per-cycle median vs frame median).
  Over onsets, vibrato, and amplitude modulation a longer fixed window
  averages in more harmonic energy, so ours runs optimistic. Both are valid
  HNR estimators with different time-frequency trade-offs; the choice is
  frozen pre-copyright. **Consequence:** VoiceMap HNR is not interchangeable
  with a Praat HNR number on running speech — compare like-for-like.
- **40 dB ceiling.** The peak clip at 0.9999 caps HNR at 40 dB; a very clean
  synthetic vowel that Praat scores ~64 dB reads 40 here. Harmless
  clinically (HNR > 20 dB is already "very clean"), but the cap means HNR is
  not a faithful measure of *how* clean above 40 dB.
- **Pitch search `[60, 400] Hz`.** Above ~400 Hz F0 the first autocorrelation
  peak can fall outside the search lag window; HNR is not characterised for
  very high sung pitches (the VRP still maps them via Clarity).
