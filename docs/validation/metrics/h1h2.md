# H1-H2 / H1-H3 — harmonic amplitude differences

> Level of the fundamental (H1) minus the 2nd / 3rd harmonic (H2 / H3), in dB.
> A glottal-source / open-quotient correlate: breathy → large positive H1-H2,
> pressed → near zero or negative.

## 1. Implementation
- File: `voicemap/metrics.py` → `HarmonicDiffCalculator` (lines 1674–1711).
- Per cycle: `_compute_cycle_dft(voice, idx, 3)` → first three harmonic
  magnitudes; `H1-H2 = 20·log10(|X1|/|X2|)`, `H1-H3 = 20·log10(|X1|/|X3|)`.
  Output clipped to **±40 dB** (a near-zero harmonic would otherwise give ±∞
  and destroy the per-cell mean).

## 2. Reference Standard
- **Iseli & Alwan (2004)**, *"An improved correction formula for the estimation
  of harmonic magnitudes…"*, ICASSP; canonical tool **VoiceSauce** (Shue et
  al. 2011).
- **Important:** VoiceMap reports the **uncorrected** harmonic ratio. Iseli &
  Alwan / VoiceSauce additionally apply a **vocal-tract (formant) correction**
  (`H1*-H2*`). On pure tones (no formants) the two coincide, so the synthetic
  GT validates the raw ratio; on real voice ours differs from a corrected value.

## 3. Test Signals
Synthetic 3-harmonic tones (in-harness, F0 = 200 Hz) with known amplitudes
`(A1, A2, A3)` ∈ {(1,.5,.25), (1,1,1), (1,2,4), (1,.7,.3)}, plus a near-zero
2nd harmonic to exercise the clip.

## 7. Known Limitations
- **Uncorrected ratio.** No vocal-tract correction, so on real voice ours is
  *not* comparable to a VoiceSauce `H1*-H2*`; near a formant the raw ratio is
  biased by the tract transfer function. For source-only comparison this is
  fine; for cross-tool comparison, correct first.
- **H1-H3 is looser** (≈ 0.2 dB vs ≈ 0.05 dB for H1-H2) — the 3rd harmonic is
  weaker and more affected by per-cycle DFT leakage when cycle boundaries fall
  between samples.
- **±40 dB clip** means an extreme ratio reads exactly ±40, not its true large
  value; intentional, but a clipped value is "≥ 40 dB", not a measurement.
- Reuses the per-cycle DFT — accuracy depends on correct cycle marking.
