# Spectral moments — centroid / bandwidth / rolloff / flatness / slope

> Frame-level shape descriptors of the magnitude spectrum. Centroid =
> "brightness"; bandwidth = spread; rolloff = 85 %-energy frequency;
> flatness = tonal-vs-noisy; slope = overall spectral tilt.

## 1. Implementation
- File: `voicemap/metrics.py` → `SpectralMomentsCalculator` (lines 698–825).
- One shared STFT per recording: 25 ms frames, 10 ms hop, per-frame zero-mean,
  Hann window, `nfft = nextpow2(2·win)`; `mag = |rfft|`, `psd = mag²`.
- Formulas (`freqs = rfftfreq(nfft, 1/sr)`):
  - `centroid = Σ psd·f / Σ psd` (power-weighted);
  - `bandwidth = √(Σ psd·(f−centroid)² / Σ psd)`;
  - `rolloff85 = min f : cumsum(psd) ≥ 0.85·Σ psd`;
  - `flatness = geomean(mag) / mean(mag)`;
  - `slope = least-squares slope of log10(mag) vs f over 0–5 kHz`.
- (The same class also emits skewness, kurtosis, alpha, hammarberg, rms —
  alpha/hammarberg are validated separately once OpenSMILE is available.)

## 2. Reference Standard
- **librosa** `feature.spectral_centroid / spectral_bandwidth /
  spectral_rolloff / spectral_flatness` (McFee et al., *librosa*, 2015) — the
  de-facto reference implementations.
- **Key method:** to test the *formula* (not librosa's different default
  framing), the SAME spectrogram our calculator builds is fed to librosa via
  its `S=` argument, with `freq=` set to our bin frequencies. Then any
  difference is purely the formula. (This mirrors how `jitter.md` feeds Praat
  the same cycle marks.)
- librosa has **no** `spectral_slope`, so slope is validated by an analytic
  ground truth (a spectrum whose `log10|S|` is exactly linear in `f`).

  flatness.
- **(B) Ground truth:**
  - slope recovers an analytic linear `log10|S|` slope (`rtol ≤ 1e-6`);
  - a 1 kHz tone reads `centroid = 1000 ± 10 Hz` through the *shipped*
    `SpectralMomentsCalculator` (ties framing + formula + cycle assignment to
    a known answer);
  - white-noise flatness `> 0.5` ≫ tone flatness `≈ 0`.

## 7. Known Limitations
- **Power-weighted, not magnitude-weighted.** Centroid/bandwidth/rolloff use
  `psd = mag²`; librosa's *defaults* use magnitude. The numbers therefore only
  match librosa when librosa is given `S = psd` (as the harness does). A user
  comparing against a librosa default call will see a (consistent) offset —
  the convention is power-weighting, documented here.
- **Framing is VoiceMap's own** (25 ms / 10 ms / Hann / per-frame zero-mean /
  `nextpow2` FFT). Absolute values depend on it; only the *formula* is
  librosa-identical. A different frame length shifts every moment.
- **Slope units** are `log10|S|` per Hz over 0–5 kHz (≈ −1e-3…0 for voice),
  not dB/octave; do not compare to dB/oct tilt figures without converting.
- **skewness/kurtosis are clipped** (`±100`, `−100…1000`) and bandwidth-floored
  at 50 Hz to stop tonal frames blowing up — they are display-bounded, not
  raw moments. Not part of this metric's acceptance (descriptive only).
