# F0 (pitch) + Clarity

> Two distinct subsystems share this file because they answer the same
> question ("what is the fundamental period?") with different algorithms for
> different consumers:
>
> - **Cycle-marker F0** — a native reimplementation of **Praat Sound_to_Pitch
>   (AC)** (`praat_pitch.py`) whose pulse marks drive jitter / shimmer. The
>   jitter and shimmer docs defer their *cycle-marker fidelity* here.
> - **VRP Clarity + MIDI / F0_Hz** — the values that ship in the Voice Range
>   Profile, produced by a **Tartini-style NSDF** (`ClarityCalculator`),
>   *not* Praat. Clarity is the NSDF peak strength (≈ 1 for a clean periodic
>   voice); MIDI/F0_Hz is the NSDF peak lag.

## 1. Implementation
**Part 1 — cycle-marker F0 (Praat AC):**
- File: `voicemap/praat_pitch.py` — `sound_to_pitch` (full pipeline:
  autocorrelation → sinc-70 peak refinement → Viterbi path), plus
  `autocorrelation_frame`, `find_candidates_parabolic`,
  `improve_maximum_sinc`, `sinc_interpolate_raised_cosine`.
- Cycle marks: `voicemap/praat_perturbation.py` →
  `sound_pitch_to_pointprocess_cc` (Praat PointProcess (cc)).

**Part 2 — VRP Clarity + MIDI (Tartini NSDF):**
- File: `voicemap/metrics.py` → `ClarityCalculator` (lines 233–391), helpers
  `_key_maxima` (lines 161–175) and `_repair_subharmonic_octaves`
  (lines 178–230).
- Pipeline: 30 Hz Butterworth HPF → leaky integrator (`a=0.995`) →
  2048-pt windows, hop 1024 → McLeod-Wyvill NSDF `2·m(τ)/n'(τ)` →
  global-argmax with iterative half-lag octave correction → low-pitch
  McLeod-Wyvill fallback → temporal-continuity subharmonic repair →
  ±12-semitone outlier clip → per-cycle assignment.
- Output: `cycle_midi`, `cycle_clarity`. `F0_Hz = 440·2^((MIDI-69)/12)`.
- Designed lag range: `[sr/1000, sr/50]` → ≈ **50–1000 Hz**, with the
  low-pitch fallback threshold at `sr/78 ≈ 78 Hz` (MIDI 39).

## 2. Reference Standard
**Part 1:** Praat (Boersma & Weenink), `fon/Sound_to_Pitch.cpp`
(autocorrelation method; Boersma 1993, *"Accurate short-term analysis of the
fundamental frequency…"*, IFA Proceedings 17) and `Sound_PointProcess.cpp`
(PointProcess (cc)). Compared against `parselmouth` `to_pitch_ac` /
`To PointProcess (cc)`.

**Part 2:** McLeod & Wyvill (2005), *"A Smarter Way to Find Pitch"*, ICMC —
the Normalised Square Difference Function (NSDF) and its "clarity" measure.
VoiceMap's constants (FFT size, integrator, HPF) match the SuperCollider
sc3-plugins `Tartini.cpp` UGen used by the project's predecessor. There is
**no Praat equivalent** for Clarity; Clarity is the NSDF peak height, not
Praat's pitch strength.

Definitions:
```
NSDF(τ)   = 2·Σ x[j]·x[j+τ]  /  Σ (x[j]² + x[j+τ]²)        ∈ [-1, 1]
clarity   = NSDF at the chosen pitch lag                    (≈1 = clean)
F0        = sr / τ*            (τ* = selected key-maximum lag)
MIDI      = 12·log2(F0 / 440) + 69
```

## 4. Validation Method
- **(A) Numerical parity — Part 1 (cycle-marker F0 vs Praat AC).** On the
  real 10 s voice, sample our `sound_to_pitch` and Praat `to_pitch_ac` on a
  common 400-point grid:
  - voicing-classification agreement ≥ 95 %;
  - median |ΔF0|/F0 ≤ 0.5 %, P90 ≤ 5 % over jointly-voiced frames;
  - cycle marks vs Praat PointProcess (cc): count ratio ∈ [0.95, 1.05],
    nearest-mark offset median ≤ 0.5 ms, P90 ≤ 2 ms.
- **(B) Synthetic ground truth — Part 2 (NSDF F0/Clarity, octave stress).**
  Run the shipped `ClarityCalculator` on each in-range synthetic vowel:
  - median in-range F0 within **±50 cents** (a quarter-tone) of the known
    F0 — i.e. *no octave error* — for 200 Hz, 800 Hz, breathy, vibrato;
  - clean-vowel clarity ≥ 0.95; breathy clarity < clean (noise lowers it);
  - silence → 0 voiced cycles.
- **(C) Real-corpus behaviour** — deferred to the corpus phase.

## 7. Known Limitations
- **NSDF low-pitch floor ≈ 78 Hz (MIDI 39).** The Tartini fallback that
  rescues octave-*down* subharmonic errors assumes any lag past `sr/78`
  (F0 < ~78 Hz) is a subharmonic and forces the pitch up. A *genuine* 70 Hz
  voice is therefore mis-tracked to a higher octave (`vowel_low_pitch_70Hz`
  reads ≈ 1002 Hz, clarity 0.75). This is below the designed VRP singing
  range (≈ D♯2); the product targets sung voice, so it is scoped out, not
  fixed (algorithm change is frozen pre-copyright and would perturb the
  48-PASS baseline). Surfaced here so sub-78 Hz F0 is read with caution.
- **Clarity is not a voicing gate.** A pure aperiodic chirp false-locks the
  NSDF (`chirp_50_1000Hz` → ≈ 668 Hz, clarity 0.999). Real voice is never a
  swept sinusoid; the VRP also gates cells by clarity ≥ 0.96 *and* SPL, so
  this pathological input is not representative. Documented, not gated.
- **Two F0 numbers exist.** The cycle-marker F0 (Praat AC) and the VRP
  MIDI/F0_Hz (NSDF) are computed by different algorithms and can differ by a
  fraction of a percent. Jitter/shimmer use the cycle-marker; the map uses
  the NSDF. This is by design (NSDF clarity is the map's quality axis).
- **Systematic +4 cent NSDF bias** at 200/800 Hz from lag quantisation +
  the leaky integrator — well within the ±50-cent (quarter-tone) bar, noted
  so the offset is not mistaken for an error.
