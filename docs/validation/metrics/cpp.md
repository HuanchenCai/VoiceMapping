# CPP / CPPS (Cepstral Peak Prominence)

> Prominence of the cepstral peak at the fundamental quefrency above the
> cepstral baseline — the single best acoustic correlate of overall dysphonia
> / breathiness, robust because it needs no F0 tracking.

## 1. Implementation
- File: `voicemap/metrics.py` → `CPPCalculator` (lines 403–466).
- SuperCollider `Cepstrum` UGen convention (the project's predecessor):
  per 100 ms window (10 ms hop, Hann), `rfft` of 2048 → **natural-log**
  magnitude → 1024-pt IFFT of the first 1024 log-mag bins → `|cepstrum|`.
  Quefrency-to-frequency mapping `f = SR/(2q)`; search bins `[25, 367]`
  ↔ `[882, 60] Hz`. Peak prominence (`_peak_prominence_batch`): cepstrum →
  dB, least-squares line over the search band, `CPP = max residual above the
  regression line`. Each cycle takes its enclosing window's CPP.
- A tiny `cpp_dither_amp` of Gaussian noise is added before the FFT to break
  exact ties on synthetic/quantised input (perturbs CPP < 0.1 dB; see §7).

## 2. Reference Standard
- **Concept:** Hillenbrand, Cleveland & Erickson (1994) / Hillenbrand &
  Houde (1996) — cepstral peak prominence; **Praat** `PowerCepstrogram →
  Get CPPS` (Maryn et al. 2010 smoothed variant).
- **Crucial caveat:** absolute CPP is *convention-dependent* — log base
  (natural vs dB), cepstrum length, smoothing, and the regression band all
  shift the number by several dB. VoiceMap's SC-convention CPP is **not**
  numerically equal to Praat CPPS and is not meant to be. The literature
  validates CPP by its **ordering** under voice degradation, which is what we
  test. Compared against `parselmouth` `Get CPPS...` for *correlation*, not
  equality.

## 3. Test Signals
**Synthetic** (`make_signals.py`, additive white noise, seed 7):
- SNR sweep `{0, 5, 10, 15, 20, 25} dB` — the monotonicity + Praat-correlation
  set (generated in-harness).
- clean modal 200 Hz (no noise) — the "clean ≫ degraded" anchor +
  reproducibility (dither) probe.
- `silent_5s.wav` — boundary (no CPP).

**Note:** `chirp_50_1000Hz.wav` is *not* used as a "low CPP" case — a linear
chirp is locally quasi-periodic and reads ~11 dB (see §7).

## 4. Validation Method
- **(A) Correlation parity** vs Praat `Get CPPS` across the 6-point SNR sweep:
  Pearson `r(CPP_ours, CPPS_Praat) ≥ 0.95`. (Absolute equality is not a
  criterion — see §2.)
- **(B) Synthetic behaviour / ground truth:**
  - monotonic in SNR: `r(CPP, SNR) ≥ 0.95`;
  - clean ≫ degraded: `CPP_clean − CPP_SNR0 ≥ 5 dB`;
  - reproducible despite dither: std over 5 runs `≤ 0.5 dB`;
  - silence → no CPP (NaN/empty).
- **(C) Real-corpus behaviour** — deferred to the corpus phase (the
  health-vs-pathology CPP separation that CPP is prized for needs Saarbruecken).

Reproducible steps:
```bash
python scripts/validate_metric.py cpp
```

## 7. Known Limitations
- **Not numerically comparable to Praat CPPS.** Ours runs several dB *lower*
  (clean 23 vs Praat 33; SNR-15 12.6 vs 15.2) because of the SC natural-log /
  1024-pt-IFFT convention. The *correlation* is r ≈ 0.98 and the ordering is
  preserved, but a VoiceMap CPP number must not be read against a Praat CPPS
  threshold. Clinical cut-offs must be re-derived on VoiceMap's own scale.
- **Flat SNR slope.** CPP rises only ~0.14 dB per dB of SNR (10.9 dB at SNR-0
  → 14.3 dB at SNR-25), so CPP compresses the noise axis far more than HNR
  (which is ~1:1). Use HNR, not CPP, when an absolute noise level is wanted;
  use CPP for robust ordering without F0 tracking.
- **Tie-breaking dither makes CPP weakly stochastic.** Run-to-run std is
  ~0.03 dB (ptp < 0.1 dB over 10 runs) — negligible clinically but it means
  CPP is not bit-exact reproducible. Acceptable; documented for the
  reproducibility goal (everything else in the pipeline is deterministic).
- **CPP is a periodicity index, not a voicing gate.** A linear chirp reads
  ~11 dB (locally quasi-periodic). The VRP gates cells by Clarity + SPL, so
  this does not leak into the map.
