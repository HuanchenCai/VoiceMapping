# Jitter (local / RAP / PPQ5 / DDP / local-absolute)

> Cycle-to-cycle variation of the glottal **period**. Clinical frequency-
> perturbation family, MDVP / Praat conventions.

## 2. Reference Standard
- **Praat** (Boersma & Weenink), `fon/VoiceAnalysis.cpp` →
  `PointProcess_getJitter_local / _rap / _ppq5 / _ddp` and
  `PointProcess.cpp` → `PointProcess_isPeriod`, `getMeanPeriod`.
- Underlying clinical definitions: MDVP (Multi-Dimensional Voice Program);
  Baken & Orlikoff, *Clinical Measurement of Speech and Voice* (2000).
- Formulas (periods `T_i`, i = 1..N between consecutive valid marks):

  ```
  jitter_local          = mean_i |T_i - T_{i-1}|            / mean_i T_i
  jitter_local_absolute = mean_i |T_i - T_{i-1}|                         [seconds]
  jitter_rap            = mean_i |T_i - mean(T_{i-1..i+1})| / mean_i T_i
  jitter_ppq5           = mean_i |T_i - mean(T_{i-2..i+2})| / mean_i T_i
  jitter_ddp            = 3 · jitter_rap
  ```

  A period is included only if `is_period` passes: its length is in
  `[pmin, pmax]` and it is not a both-sides outlier (neighbour ratios both
  > `max_period_factor`). This gate is translated 1:1 from Praat.

- Reference implementation we compare against: `parselmouth` (Praat
  6.x bindings), `call(pp, "Get jitter (local)", 0, 0, 1e-4, 0.02, 1.3)`.

## 4. Validation Method
A metric is **cycle-marker + formula**. This file validates the **formula**
layer. The cycle-marker fidelity is validated separately in
`f0_clarity.md` (cycle count + timing offset vs Praat).

- **(A) Numerical parity** vs Praat, formula layer only.
  - Identical input: Praat's own `PointProcess (cc)` marks on the real 10 s
    voice channel are handed to *both* Praat's jitter query and our numpy
    function.
  - Tolerance: `atol=1e-9` (fraction forms), `atol=1e-12` (absolute form).
  - Acceptance: every form within tolerance.
- **(B) Synthetic ground truth**.
  - Input: analytically-known cycle marks for an alternating ±d period
    pattern (`d = jitter/2`), for which `jitter_local = 2d` exactly.
  - Expected output: the imposed 0.5 % / 2 % / 0 %.
  - Tolerance: `rtol=1e-3` (non-zero), `atol=1e-5` (zero case).
- **(C) Real-corpus behaviour** — deferred to the corpus phase; (A)+(B)
  meet the P0 *formula* bar. End-to-end corpus distribution will be added
  once Saarbruecken is wired (see `corpora/saarbruecken.md`).

## 7. Known Limitations
- **End-to-end synthetic jitter under-reports.** Praat's `PointProcess (cc)`
  cycle-marker smooths a sub-sample alternating period toward a regular
  grid: an imposed 2 % alternating jitter is *detected* as ~0.22 % once the
  marker re-locks the pulses. This is a property of the cycle-**marker**
  (the F0 / cc stage), not the jitter **formula** — which recovers the
  imposed value exactly when given the true marks (§5 part B). Real, more
  slowly-varying jitter is not affected the same way; the parity check (§5
  part A) on real marks is exact.
- RAP / PPQ5 of a perfectly-alternating sequence equal `(2/3)·local` and
  `(2/5)·local` respectively (smoothing windows of 3 and 5). Expected, not
  a defect — documented so the synthetic numbers are not misread.
- Period-validity gate (`max_period_factor=1.3`) can drop cycles at
  octave jumps / voicing onsets; cycle counts therefore vary slightly
  between implementations (≤5 %, see `conventions.md` §1).
- Not characterised below F0 = 60 Hz (outside `[pmin,pmax]` for `pmax=0.02`).
