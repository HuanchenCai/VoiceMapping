# CSE — Cycle Sample Entropy

> Sample-entropy of the cycle-to-cycle variation of the EGG harmonic
> structure (amplitude + phase). A nonlinear regularity measure — irregular /
> disordered phonation has higher entropy.

## 1. Implementation
- File: `voicemap/metrics.py`
  - `_batch_sample_entropy_m1` (lines 85–102): the core **m = 1 Sample
    Entropy** — `−ln(A/B)`, where `B` = count of length-1 template pairs
    within tolerance `r` and `A` = the subset also matching at the next sample
    (length-2). Vectorised over a batch of windows.
  - `EntropyCalculator` (lines 614–657): per EGG cycle, DFT → harmonic
    amplitudes (in Bel, `2·log10|·|`) and phases; for each harmonic a sliding
    window (amplitude / phase window sizes from config) → `_batch_sample_
    entropy_m1` → summed across harmonics into one entropy per cycle.

## 2. Reference Standard
- **Richman & Moorman (2000)**, *"Physiological time-series analysis using
  approximate entropy and sample entropy"*, Am. J. Physiol. — the Sample
  Entropy definition.
- Reference tool: **`nolds.sampen`** (`emb_dim=1`, matching tolerance `r`).
  Pinned `nolds==0.5.2` in `requirements-validation.txt` (0.6.x crashes on
  import under Python 3.11 — eager dataset load).

## 3. Test Signals
Synthetic 1-D sequences (in-harness, seeded): white noise (N = 60, 100),
a sine, an AR(1) process, and a deterministic ramp. `r = 0.2·std` (the
conventional tolerance).

## 4. Validation Method
- **(A) Numerical parity** — `_batch_sample_entropy_m1(seq, r)` vs
  `nolds.sampen(seq, emb_dim=1, tolerance=r)`, `atol = 1e-9`, across noise /
  sine / AR(1). (Validates the entropy *formula*; the DFT-harmonic windowing
  is the metric-specific wrapper around it.)
- **(B) Behavioural ground truth** — entropy increases with disorder: a
  deterministic ramp (≈ 0) < a periodic sine < white noise.
- **(C)** corpus deferred.

Reproducible steps:
```bash
pip install nolds==0.5.2
python scripts/validate_metric.py cse
```

## 7. Known Limitations
- **Only the entropy primitive is parity-checked.** The full CSE = Σ over
  harmonics of `sampen(window of harmonic-amplitude-in-Bel)` +
  `sampen(window of |phase|)`. The DFT extraction, the Bel scaling, the
  per-harmonic windowing and the summation are VoiceMap conventions (matching
  the project's SC predecessor), not independently parity-checked here — they
  have no external reference tool. The *building block* (Sample Entropy) is
  exact; the assembly is a design choice.
- **m = 1 only.** Many SampEn studies use m = 2; ours is fixed at m = 1 (the
  primitive is named `_m1`). Values are not comparable to m = 2 literature.
- **Tolerance `r` is config-set per channel** (amplitude / phase) rather than
  the textbook `0.2·std`; entropy magnitude depends on `r`, so absolute CSE is
  VoiceMap-scale, validated by ordering not by an absolute number.
- **Needs ≥ window-size voiced cycles**; short segments return 0.
