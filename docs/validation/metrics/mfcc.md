# MFCC 1–13 (mel-frequency cepstral coefficients)

> The standard timbral feature vector for speech/voice ML. 13 coefficients
> per frame summarising the mel-scaled spectral envelope.

## 1. Implementation
- File: `voicemap/metrics.py` → `MFCCCalculator` (lines 892–956),
  `_build_mel_filterbank` (lines 873–889), `_hz_to_mel`/`_mel_to_hz`
  (lines 865–870, HTK formula `2595·log10(1+f/700)`).
- Pipeline: pre-emphasis `0.97` → 25 ms / 10 ms frames, per-frame zero-mean,
  **Hamming** → `nfft = nextpow2(2·win)` power spectrum → **26-band HTK mel
  filterbank** → **natural log** → **DCT-II (ortho)** → first **13**.
- Filterbank: triangles with vertices quantised to FFT bins
  (`floor((nfft+1)·hz/sr)`) — the "HTK-tutorial" convention.

## 2. Reference Standard
- **librosa** `feature.mfcc` (McFee et al. 2015), the de-facto reference, and
  the HTK MFCC definition (Young et al., *HTK Book*).
- Two deliberate convention choices make ours *not bit-identical* to a default
  librosa call, both standard and both immaterial to the cepstrum shape:
  1. **vertex-quantised** filterbank (HTK tutorial) vs librosa's continuous
     triangles;
  2. **natural log** vs librosa's dB (`10·log10`) — a constant `ln(10)/10 ≈
     0.23` scale on every coefficient.
- Validation is therefore **layered**: mel centres and the DCT are checked for
  *exact* equality; the full MFCC is checked by *correlation* (scale-invariant).

## 4. Validation Method
- **(A) Layered parity vs librosa:**
  - mel centre frequencies == `librosa.mel_frequencies(htk=True)` (`≤ 1e-6 Hz`);
  - DCT-II ortho == `librosa.feature.mfcc(S=log_mel)` on the **same** log-mel
    (`≤ 1e-9`);
  - filterbank weights ≈ `librosa.filters.mel(htk=True, norm=None)`
    (`max|Δw| ≤ 0.2`, the vertex-quantisation gap, ~0.11);
  - full MFCC per-coefficient Pearson `r ≥ 0.99` vs the librosa-filterbank
    path (min over the 13 coeffs ≈ 0.999).
- **(B) Shipped-calculator sanity:** `MFCCCalculator` emits 13 finite
  coefficients whose `mfcc1` mean matches the inline computation (`≤ 1.0`).
- **(C) Real-corpus behaviour** — deferred to the corpus phase.

## 7. Known Limitations
- **Not numerically equal to a default librosa MFCC.** Two consistent offsets:
  (1) the natural-log-vs-dB constant scale (`×0.23`); (2) the vertex-quantised
  filterbank (r ≥ 0.999 but not bit-equal). For ML this is irrelevant —
  features are standardised anyway — but a user must not expect ours and a raw
  `librosa.feature.mfcc()` to print the same numbers. Match conventions first.
- **No liftering, no C0, no deltas.** Coefficients are MFCC **1–13** (not 0–12);
  C0 (energy) is excluded and Δ/ΔΔ are not produced. Add them downstream if a
  model needs them.
- **Natural-log floor `1e-15`** can clamp silent mel bands, biasing MFCCs of
  near-silent frames; gate on voicing/SPL before using MFCCs clinically.
- Shares the frame/`nextpow2` framing of the other spectral metrics — see
  `spectral_moments.md §7`.
