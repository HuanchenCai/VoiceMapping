# Qcontact / dEGGmax / Icontact — VoiceMap signature EGG metrics

> VoiceMap's **own amplitude-based** EGG descriptors. **Not** the standard
> time-based contact quotient (that is `OQ`, see `oq.md`). Fixed as an
> **identity marker** — the formula is deliberately VoiceMap-specific and is
> not to be changed (CLAUDE.md §11).

## 1. Implementation
- File: `voicemap/metrics.py` → `QcontactCalculator` (lines 560–604).
- Per cycle (EGG `cyc`, `cmin/cmax`, period `T = ticks`):
  - `Qcontact = cmin / (cmin − cmax)`  — amplitude baseline ratio (the code
    names it `integral`);
  - `amp_sc = 1 / (p2p · (−0.5) · sin(2π/T))`, `p2p = cmin − cmax`;
  - `delta = max(EGG[s]−EGG[s−1], max(diff(cyc)))` — closure step / steepest
    rise; `dEGGmax = delta · amp_sc`;
  - `Icontact = log10(max(dEGGmax, 1)) · Qcontact`.

## 2. Reference Standard
- Conceptually Howard (1995) contact-quotient family + the project's
  SuperCollider predecessor. **However** the shipped formula is VoiceMap's own
  amplitude-based definition, *not* a time-based CQ, and is an **identity
  marker** (CLAUDE.md §11 — "改 Qcontact/Icontact 公式 → 拒绝，是身份标识").
- No SC source is available to parity against, and the formula intentionally
  differs from any external definition, so validation is **implementation-
  level** (the code evaluates its formula exactly) plus a closed form for
  `dEGGmax`.

## 7. Known Limitations
- **Not a standard contact quotient.** `Qcontact = cmin/(cmin−cmax)` depends
  ONLY on the min/max amplitudes, **not on contact time**, and is **negative**
  for a positive-baseline EGG (e.g. −0.25 for `[0.2, 1.0]`). It must not be
  read as a Howard/Baken time-based CQ — use **OQ** (`oq.md`) for that. This is
  intentional: Qcontact/Icontact are VoiceMap's signature axes (identity
  marker), frozen by project rule. The validation confirms the code is
  faithful to the formula, **not** that the formula is a textbook CQ.
- **dEGGmax is period-dominated** for an idealised closure: `2/sin(2π/T) ≈ T/π`,
  so it scales with the period (≈ inversely with F0) rather than with contact
  speed, once the closure is a full-amplitude step. On a real, finite-slope
  EGG it reflects the steepest rise; the synthetic closed form is the clean
  anchor, not the typical real value.
- **No SC parity.** Without the SuperCollider source the original intent of the
  `integral`/`amp_sc` terms cannot be cross-checked; if paper-grade provenance
  is needed, recover the SC source and add a parity test. Flagged for the
  methodology write-up.
- **Icontact inherits** Qcontact's sign/scale caveats (it multiplies by it).
