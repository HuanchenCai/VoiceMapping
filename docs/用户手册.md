## 1. 软件概述

本软件用于嗓音的多维度声学/电声门图（EGG）分析。给定立体声 WAV 文件
（通道 1 = 麦克风，通道 2 = EGG），软件对录音逐周期切分，在 (MIDI 音高,
SPL 声压级) 二维网格上聚合 40+ 项嗓音质量指标，生成 Voice Range Profile
(VRP) 热图，可用于嗓音诊断、歌手训练、声乐教学和科研。

**主要功能**

- 单文件 / 多文件录音管理
- 40+ 嗓音质量指标计算（CPP、HNR、Jitter、Shimmer、Qcontact、聚类等）
- VRP 热图实时渲染 + 鼠标 hover 实时显示当前 cell 数值
- 临床范围参考（基于 MDVP / 文献阈值）
- 多种格式导出（CSV、Excel 多 sheet、PDF 报告）
- 中英文双语切换
- 多 wav 联合 K-means 中心点训练，跨录音保持簇标签一致

## 2. 系统要求

- **操作系统**：Windows 10 / 11（64 位）
- **CPU**：x86_64，2 核以上
- **内存**：≥ 4 GB（分析 60 秒立体声 wav 约用 800 MB）
- **磁盘**：≥ 200 MB（程序 + 缓存）
- **可选**：tkinterdnd2 提供原生拖放（已打包，无需额外安装）

## 3. 启动

### 3.1 双击 `启动.bat`

项目根目录有 `启动.bat`，双击即打开 GUI。

### 3.2 命令行

```
python main.py --gui              # 打开图形界面
python main.py audio\test.wav     # 命令行直接分析
python main.py --help             # 查看所有选项
```

### 3.3 安装包

发布版双击 `VoiceMap_v1.0.0_setup.exe` 即可安装，安装完成后
桌面 / 开始菜单生成快捷方式，双击启动。

### 4.1 菜单栏

- **文件**：打开 WAV / 打开输出目录 / 导出 Excel / 生成报告 / 对比 2 段录音 / 设置 / 退出
- **编辑**：标注（点 cell 加注释）/ 复位标注 / 复制图片 / 保存图片
- **参数**：上/下一个 metric / 5 大分类 cascade（声学/EGG/唱歌特异性/聚类/密度）/ 聚类中心 (Centroid) 子菜单
- **视图**：拟合曲线（音域中心线 / metric 趋势线）/ 日志面板
- **帮助**：关于 / 语言（中文 / English）

### 4.2 Tracks Panel（录音轨）

- 拖入或菜单 文件 → 打开 WAV 加载录音
- 右侧显示当前状态指示（● 就绪 / 分析中… / 完成 · N 点）和"开始分析"按钮
- 加载完文件需点 "开始分析" 按钮才开始分析（不自动触发）
- 多 wav 同时拖入：全部进入列表，按钮分析当前 active 文件
- 点击任意行切换为 active 文件（已分析的从内存恢复，未分析的就地分析）
- 行内显示：编号、状态符号（○ 待分析 / ⏵ 分析中 / ✓ 已分析 / ✗ 失败）、
  文件名、采样率、时长、通道数、网格数、Unicode block 波形

### 4.3 Metric Bar

显示当前 metric 名（如 `CPP`）。切换方式：

1. 顶部菜单 参数 → 分类 → metric
2. 键盘 ← / → 上下切换
3. 画布两侧浮动 ◀ ▶ 箭头点击

### 4.4 Inspector（右侧详情栏，固定 420 px）

- **顶部**：当前 metric 大字 + ⓘ 提示图标 + 描述 + 单位
  - 鼠标 hover 标题或 ⓘ 显示详细公式 / 临床意义
- **中部（可滚动）**：参考范围 —— 4 档严重度色（绿 / 灰 / 橙 / 红），
  按指标自动加载对应文献阈值
- **底部（固定）**：本次值卡 —— 鼠标移到画布任意 cell，实时显示
  该 cell 的 (MIDI, SPL) 坐标 + metric 值 + 严重度色 + 严重度文字
- 导出 Excel / 生成报告 / 对比录音 等动作走顶部菜单（文件）

### 4.5 Status Bar

显示当前 active 文件的元信息：文件名 · 网格数 · k 值 · 周期数 · 耗时。
右侧固定版权 + 版本号。

## 5. 操作流程

### 5.1 单文件分析

1. 双击 `启动.bat` 打开 GUI
2. 拖一个立体声 WAV 进入窗口（或菜单 文件 → 打开 WAV）
3. 等待分析（约 12s @ 60s 录音）—— 进度对话框实时显示阶段
4. 完成后右侧 VRP 热图自动渲染当前 metric（默认 CPP）
5. 鼠标移到热图任意位置 → Inspector 实时显示该 cell 的数值 + 临床档位
6. 顶部 参数 菜单切换其他 metric / 键盘 ←→ 快速切

### 5.2 多文件管理

1. 同时拖入多个 WAV → 全部进入 Tracks Panel，第一个自动分析
2. 点击 Tracks 列表里的其他行 → 切换 active 文件，自动分析或恢复缓存
3. 右上 Inspector 自动跟随显示当前 active 文件的指标详情

### 5.3 导出

- **Excel**：菜单 文件 → 导出 Excel → 输出 .xlsx，含 Summary + Grouped +
  每个 metric 一个 heatmap pivot sheet
- **报告**：菜单 文件 → 生成报告 → 输出 .md 临床叙述报告，
  按 6 大类（总览 / 嗓音质量 / EGG / 唱歌 / 频谱 / 自动观察）
- **图片**：菜单 编辑 → 复制图片（剪贴板）/ 保存图片（PNG/PDF/SVG/EMF）

### 5.4 对比两段录音

1. 菜单 文件 → 对比 2 段录音
2. 在弹窗中加载 A、B 两个 VRP CSV
3. 选 metric → 渲染 A | B | A−B 三联图
4. 导出 PNG 留档

### 5.5 跨录音保持聚类标签一致

1. 选一段代表性 wav 分析
2. 菜单 参数 → 聚类中心 → 保存当前 centroids → 输出 cEGG.csv
3. 之后分析其他 wav 前：菜单 参数 → 聚类中心 → 加载 centroid CSV
4. 后续分析自动跳过 K-means 训练，直接分类，保证 cluster 1..5 在不同
   录音里语义一致

## 6. 设置

菜单 文件 → 设置：

- **Clarity 阈值**：0.80–1.00。McLeod-Wyvill NSDF 音高检测置信度下限。
  低于该值的周期被丢弃。默认 0.96
- **聚类簇数 k**：2–10，默认 5。EGG 波形 K-means / 嗓音质量 cPhon
  K-means 共用
- **谐波数 n**：3–20，默认 10。EGG 聚类的特征向量维度（每周期取前 n
  个谐波的幅度+相位）
- **输出目录**：CSV / Excel / 报告 / PNG 默认输出位置
- **自动导出 PNG**：分析完成后自动批量导出每个 metric 的 PNG 到
  `<output>/plots/`

## 7. 关键指标速查

下表列出常用指标的临床/学术阈值（来源：MDVP / KayPENTAX / 文献）。
完整阈值表见软件 Inspector 临床参考范围卡。

| 指标 | 健康范围 | 注意 / 病理 | 来源 |
|------|----------|--------------|------|
| CPP | ≥ 14 dB | < 8 dB（可能气声） | Hillenbrand 1996 |
| HNR | 20–35 dB | < 10 dB（可能病理） | MDVP |
| Jitter | < 1.04% | > 2%（病理） | MDVP |
| Shimmer | < 3.81% | > 6%（病理） | MDVP |
| Qcontact | 0.3–0.6 | — | FonaDyn / SC |
| VibratoRate | 5–7 Hz（典型） | > 10 Hz（异常颤动） | 学界共识 |

**注**：Qcontact / Icontact / HRFegg / Entropy 等 EGG 指标的具体公式
来自 KTH FonaDyn (SuperCollider) 项目，本软件保持完全一致。

## 8. 命令行用法

GUI 之外，所有功能都可命令行调用，便于批处理：

```bash
# 单文件
python main.py audio/test.wav

# 自定义 clarity 阈值 + 簇数 + 合并图
python main.py audio/test.wav --clarity 0.97 --cluster-k 6 \
    --plot-mode combined

# 跨录音保持簇标签一致
python main.py refA.wav --save-centroids cEGG.csv
python main.py testB.wav --load-centroids cEGG.csv

# 联合训练（多 wav 一起算 centroid）
python main.py --train-centroids out.csv a.wav b.wav c.wav

# 批处理整个目录
python main.py --batch audio/ --plot-mode none --load-centroids cEGG.csv

# 导出 Excel + 报告
python main.py audio/test.wav --excel --report

# 对比两个已存在的 VRP CSV
python main.py --compare a.csv b.csv --compare-metric CPP
```

## 9. 故障排查

| 现象 | 原因 / 解决 |
|------|-------------|
| 启动后窗口标题是空的 | tkinterdnd2 未安装。GUI 可用，仅拖放退化为点击 |
| 第一次分析特别慢 | numba JIT 冷启动，第二次起恢复正常 |
| Excel 导出失败 | 检查 openpyxl 是否安装；输出目录是否可写 |
| 鼠标 hover Inspector 不更新 | 可能在标注模式（编辑 → 标注 ●）。再点一下"标注"切回 |
| 切语言后菜单消失一下 | 正常 —— 菜单条整体重建，毫秒级 |
| 多 wav 分析时第二个不自动开始 | 设计如此。点 Tracks 行触发 |
